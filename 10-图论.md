## 200. 岛屿数量（Number of Islands）· Medium

🔗 https://leetcode.cn/problems/number-of-islands/

### 题目大意
`'1'` 是陆地、`'0'` 是水，上下左右相连的陆地组成岛屿，求岛屿数量。

### 思路
**DFS 扫连通块**。遍历每个格子，遇到没访问过的 `'1'`：岛屿数 +1，然后从它出发 DFS，把整座相连的陆地全部"淹没"（标记为 `'0'`），避免重复计数。每次新发现的 `'1'` 都对应一座新岛。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)（递归栈最坏）。

### 代码
```java
class Solution {
    public int numIslands(char[][] grid) {
        int m = grid.length, n = grid[0].length;
        int count = 0;
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == '1') {  // 发现新岛
                    count++;
                    dfs(grid, i, j);      // 淹没整座岛
                }
            }
        }
        return count;
    }

    // 把 (i,j) 所在的连片陆地全部置为 '0'
    private void dfs(char[][] grid, int i, int j) {
        // 越界 或 不是陆地 → 停止
        if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length || grid[i][j] != '1')
            return;
        grid[i][j] = '0';            // 标记已访问（淹没）
        dfs(grid, i + 1, j);         // 下
        dfs(grid, i - 1, j);         // 上
        dfs(grid, i, j + 1);         // 右
        dfs(grid, i, j - 1);         // 左
    }
}
```

### 小结 / 语法点
- **连通块计数**：外层找未访问的起点，每找到一个就 DFS 淹没整块。
- 直接改原 grid 当 visited 标记，省一个数组（若不能改原数组，另开 `boolean[][] visited`）。
- 同款可解 695 岛屿最大面积、994 腐烂橘子（BFS 版）。

## 994. 腐烂的橘子（Rotting Oranges）· Medium

🔗 https://leetcode.cn/problems/rotting-oranges/

### 题目大意
`0` 空、`1` 新鲜橘子、`2` 烂橘子。每分钟，烂橘子会让其上下左右相邻的新鲜橘子变烂。求所有橘子腐烂所需的最少分钟数；若有橘子永远烂不了，返回 `-1`。

### 思路
**多源 BFS**（同时从所有初始烂橘子扩散）。
1. 先把所有初始烂橘子入队，并统计新鲜橘子数 `fresh`。
2. 一层一层 BFS（每层 = 1 分钟），把相邻新鲜橘子变烂、入队、`fresh--`。
3. BFS 结束后，若 `fresh == 0` 返回经过的分钟数，否则返回 -1。
"按层扩散"正好对应"按分钟传播"，这是 BFS 求最短时间的典型应用。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)。

### 代码
```java
class Solution {
    public int orangesRotting(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        Queue<int[]> q = new LinkedList<>();
        int fresh = 0;

        // 初始：所有烂橘子入队，统计新鲜橘子
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 2) q.offer(new int[]{i, j});
                else if (grid[i][j] == 1) fresh++;
            }

        if (fresh == 0) return 0;        // 没有新鲜橘子，0 分钟

        int minutes = 0;
        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
        // 按层 BFS，每层代表过了一分钟
        while (!q.isEmpty() && fresh > 0) {
            minutes++;
            int size = q.size();
            for (int k = 0; k < size; k++) {
                int[] cur = q.poll();
                for (int[] d : dirs) {
                    int x = cur[0] + d[0], y = cur[1] + d[1];
                    if (x < 0 || x >= m || y < 0 || y >= n || grid[x][y] != 1) continue;
                    grid[x][y] = 2;       // 变烂
                    fresh--;
                    q.offer(new int[]{x, y});
                }
            }
        }
        return fresh == 0 ? minutes : -1; // 还有新鲜的没烂到 → -1
    }
}
```

### 小结 / 语法点
- **多源 BFS**：把所有起点一次性入队，再分层扩散——求"最短时间/最少步数"的标准模型。
- 用 `int size = q.size()` 分层，每层 minutes++。
- 队列元素用 `int[]{x, y}` 存坐标。

## 207. 课程表（Course Schedule）· Medium

🔗 https://leetcode.cn/problems/course-schedule/

### 题目大意
共 `numCourses` 门课，`prerequisites[i] = [a, b]` 表示学 a 前必须先学 b。判断能否修完所有课（即课程依赖图是否**无环**）。

### 思路
**拓扑排序（BFS / Kahn 算法）**。把课程看作有向图（`b → a`），判断是否存在拓扑序：
1. 建邻接表 + 统计每个节点**入度**。
2. 所有入度为 0 的节点入队（没有前置课，可以直接学）。
3. 不断出队一个节点（学一门课），把它指向的节点入度 -1；某节点入度减到 0 就入队。
4. 若最终"学完的课数 == 总课数"，说明无环、可完成；否则有环。

### 复杂度
- 时间 O(V + E)，空间 O(V + E)。

### 代码
```java
class Solution {
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        // 邻接表：graph.get(b) 里存所有"依赖 b"的课程 a
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) graph.add(new ArrayList<>());
        int[] indegree = new int[numCourses];

        for (int[] p : prerequisites) {
            graph.get(p[1]).add(p[0]); // 边 b -> a
            indegree[p[0]]++;          // a 的入度 +1
        }

        // 入度为 0 的课先入队
        Queue<Integer> q = new LinkedList<>();
        for (int i = 0; i < numCourses; i++)
            if (indegree[i] == 0) q.offer(i);

        int finished = 0;
        while (!q.isEmpty()) {
            int cur = q.poll();
            finished++;                       // 学完一门
            for (int next : graph.get(cur)) {
                if (--indegree[next] == 0) q.offer(next); // 前置都学完，可学
            }
        }
        return finished == numCourses;        // 全学完 ⇒ 无环
    }
}
```

### 小结 / 语法点
- "**判断有向图是否有环 / 求依赖顺序**" → 拓扑排序（入度 BFS）。
- 建图三件套：邻接表 `List<List<Integer>>`、入度数组、入度为 0 入队。
- 207 求能否完成；210 课程表 II 要输出顺序，把出队顺序记下来即可。

## 208. 实现 Trie（前缀树）（Implement Trie）· Medium

🔗 https://leetcode.cn/problems/implement-trie-prefix-tree/

### 题目大意
实现前缀树，支持 `insert(word)`、`search(word)`（是否存在完整单词）、`startsWith(prefix)`（是否有以 prefix 开头的单词）。

### 思路
**多叉树（每个节点 26 个孩子）**。每个节点用 `Trie[26]` 数组指向 26 个字母的子节点，外加一个布尔 `isEnd` 标记"是否有单词在此结尾"。
- `insert`：沿字符逐层下探，缺失的子节点就新建，最后一个字符处置 `isEnd = true`。
- `search`：沿字符走，走不通返回 false；走到底看 `isEnd`。
- `startsWith`：同 search，但走到底即返回 true（不看 isEnd）。

### 复杂度
- 每次操作 O(L)，L 为单词长度。空间 O(总字符数 × 26)。

### 代码
```java
class Trie {
    private Trie[] children; // 26 个字母的子节点
    private boolean isEnd;   // 是否有单词在此结尾

    public Trie() {
        children = new Trie[26];
        isEnd = false;
    }

    public void insert(String word) {
        Trie node = this;
        for (char c : word.toCharArray()) {
            int i = c - 'a';
            if (node.children[i] == null) node.children[i] = new Trie(); // 缺则建
            node = node.children[i];
        }
        node.isEnd = true;   // 标记单词结尾
    }

    public boolean search(String word) {
        Trie node = find(word);
        return node != null && node.isEnd; // 必须正好是一个单词
    }

    public boolean startsWith(String prefix) {
        return find(prefix) != null;       // 只要前缀路径存在即可
    }

    // 沿 s 的字符下探，返回终点节点；中途断了返回 null
    private Trie find(String s) {
        Trie node = this;
        for (char c : s.toCharArray()) {
            int i = c - 'a';
            if (node.children[i] == null) return null;
            node = node.children[i];
        }
        return node;
    }
}
```

### 小结 / 语法点
- Trie 节点 = `children[26]` + `isEnd`。`search` 与 `startsWith` 的唯一区别是**看不看 isEnd**。
- 把公共的下探逻辑抽成 `find`，两个查询方法复用。
- 适合"大量字符串前缀匹配/自动补全"场景。

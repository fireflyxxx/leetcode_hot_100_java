## 215. 数组中的第 K 个最大元素（Kth Largest Element）· Medium

🔗 https://leetcode.cn/problems/kth-largest-element-in-an-array/

### 题目大意
返回数组中第 `k` 大的元素（是排序后第 k 大，不是去重后）。

### 思路
**维护大小为 k 的小根堆**。遍历所有数入堆，一旦堆大小超过 k 就弹出堆顶（最小的）。这样堆里始终保留"目前最大的 k 个数"，最终堆顶（这 k 个里最小的）就是第 k 大。
（也可用快速选择 quickselect 做到平均 O(n)，但堆解法更好写好记。）

### 复杂度
- 时间 O(n log k)，空间 O(k)。

### 代码
```java
class Solution {
    public int findKthLargest(int[] nums, int k) {
        PriorityQueue<Integer> minHeap = new PriorityQueue<>(); // 小根堆
        for (int x : nums) {
            minHeap.offer(x);
            if (minHeap.size() > k) minHeap.poll(); // 超过 k 个，弹掉最小的
        }
        return minHeap.peek();   // 堆里最小的 = 第 k 大
    }
}
```

### 小结 / 语法点
- **求第 K 大用小根堆**（堆顶是 top-k 里最小的，即第 K 大）；求第 K 小则用大根堆。
- 维持堆大小为 k：`if (size > k) poll()`。

## 347. 前 K 个高频元素（Top K Frequent Elements）· Medium

🔗 https://leetcode.cn/problems/top-k-frequent-elements/

### 题目大意
返回数组中出现频率前 `k` 高的元素。

### 思路
两步：
1. **哈希表计数**：统计每个元素的出现次数。
2. **小根堆按频次取前 k**：把 `(元素, 频次)` 入堆，按频次比较，维持堆大小为 k。遍历完，堆里就是频次最高的 k 个。

### 复杂度
- 时间 O(n log k)，空间 O(n)。

### 代码
```java
class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        // 1) 计数
        Map<Integer, Integer> cnt = new HashMap<>();
        for (int x : nums) cnt.merge(x, 1, Integer::sum);

        // 2) 小根堆，按频次排序，存 int[]{元素, 频次}
        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[1] - b[1]);
        for (Map.Entry<Integer, Integer> e : cnt.entrySet()) {
            pq.offer(new int[]{e.getKey(), e.getValue()});
            if (pq.size() > k) pq.poll();         // 维持前 k 高
        }

        // 3) 取出结果
        int[] res = new int[k];
        for (int i = 0; i < k; i++) res[i] = pq.poll()[0];
        return res;
    }
}
```

### 小结 / 语法点
- **计数 + 堆**是 Top K 频次的标准两步。
- 堆元素用 `int[]{值, 频次}`，比较器按频次 `a[1]-b[1]`。
- 进阶可用**桶排序**做到 O(n)（按频次当下标分桶）。

## 295. 数据流的中位数（Find Median from Data Stream）· Hard

🔗 https://leetcode.cn/problems/find-median-from-data-stream/

### 题目大意
设计数据结构，支持不断 `addNum(num)` 添加数字，并随时 `findMedian()` 返回当前所有数的中位数。

### 思路
**对顶堆（大根堆 + 小根堆）**。
- `small`：**大根堆**，存较小的一半，堆顶是这半里的最大值。
- `large`：**小根堆**，存较大的一半，堆顶是这半里的最小值。
- 保持两堆元素个数相等或 `small` 比 `large` 多一个。
- 中位数：两堆相等 → 取两个堆顶的平均；`small` 多一个 → 取 `small` 堆顶。

addNum 时先丢进 small，再把 small 的堆顶移给 large（保证 small 的所有数 ≤ large 的所有数），最后如果 large 比 small 多就把 large 堆顶还给 small（平衡）。

### 复杂度
- `addNum` O(log n)，`findMedian` O(1)。空间 O(n)。

### 代码
```java
class MedianFinder {
    // 较小的一半：大根堆，堆顶是这半的最大值
    private PriorityQueue<Integer> small = new PriorityQueue<>((a, b) -> b - a);
    // 较大的一半：小根堆，堆顶是这半的最小值
    private PriorityQueue<Integer> large = new PriorityQueue<>();

    public MedianFinder() {}

    public void addNum(int num) {
        small.offer(num);            // 先放进较小半
        large.offer(small.poll());   // 把较小半的最大值挪到较大半（保证 small 全 <= large）
        // 若较大半更多，把它的最小值还回较小半（让 small 个数 >= large）
        if (large.size() > small.size()) small.offer(large.poll());
    }

    public double findMedian() {
        if (small.size() > large.size()) return small.peek();      // 奇数个：中间在 small 顶
        return (small.peek() + large.peek()) / 2.0;                // 偶数个：两顶平均
    }
}
```

### 小结 / 语法点
- **对顶堆**是动态中位数的标准结构：大根堆管小半、小根堆管大半。
- 入数三步：进 small → small 顶给 large → 不平衡则 large 顶还 small。
- 平均时写 `/ 2.0` 保证浮点除法（别写 `/2` 否则整数除法丢精度）。

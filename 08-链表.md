> **节点定义**（LeetCode 已给，下面代码默认存在）：
> ```java
> public class ListNode {
>     int val;
>     ListNode next;
>     ListNode() {}
>     ListNode(int val) { this.val = val; }
>     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
> }
> ```

> 💡 **三个通用心法**
> 1. **dummy 虚拟头**：`ListNode dummy = new ListNode(0, head);` 最后 `return dummy.next;`。凡是"头结点可能被删/被换"的题都用它，省去特判。
> 2. **快慢指针**：找中点、找环、找倒数第 k 个。
> 3. **改指针前先暂存 next**：`ListNode next = cur.next;`，否则断链就找不回来了。

## 160. 相交链表（Intersection of Two Linked Lists）· Easy

🔗 https://leetcode.cn/problems/intersection-of-two-linked-lists/

### 题目大意
找出两个链表**相交的起始节点**（相交即从某点起共享后续节点），不相交返回 `null`。

### 思路
**双指针走"你我之和"**。设链表 A 独有长度 a、B 独有长度 b、公共部分 c。
指针 `pa` 走 A 再接着走 B，`pb` 走 B 再接着走 A。两者各走 `a+c+b = b+c+a` 步后，必然同时到达相交点（或同时到达 null）。
直觉：把两条路拼接成等长，消除长度差，自然在交点相遇。

### 复杂度
- 时间 O(m+n)，空间 O(1)。

### 代码
```java
public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        ListNode pa = headA, pb = headB;
        // 相等时退出：要么是交点，要么都为 null（不相交）
        while (pa != pb) {
            pa = (pa == null) ? headB : pa.next; // A 走完接 B
            pb = (pb == null) ? headA : pb.next; // B 走完接 A
        }
        return pa;
    }
}
```

### 小结 / 语法点
- "消除长度差"的双指针拼接技巧，记住 `pa = pa==null ? headB : pa.next`。
- 不相交时两指针都会走到 null 而相等退出，返回 null，逻辑自洽。

## 206. 反转链表（Reverse Linked List）· Easy

🔗 https://leetcode.cn/problems/reverse-linked-list/

### 题目大意
反转单链表，返回新的头结点。`1→2→3→4→5` → `5→4→3→2→1`。

### 思路
**迭代三指针**：`prev`（已反转部分的头，初始 null）、`cur`（当前处理节点）、`next`（暂存下一个）。
每步：先存 `next`，把 `cur.next` 指回 `prev`，再 `prev`、`cur` 整体右移。`cur` 走到 null 时，`prev` 就是新头。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;   // 已反转部分的头
        ListNode cur = head;    // 当前待反转节点

        while (cur != null) {
            ListNode next = cur.next; // 1) 暂存下一个，防断链
            cur.next = prev;          // 2) 当前节点指向前驱（反转）
            prev = cur;               // 3) prev 前移
            cur = next;               // 4) cur 前移
        }
        return prev;                  // cur 到 null，prev 是新头
    }
}
```

### 递归写法（理解递归很有帮助）
```java
class Solution {
    public ListNode reverseList(ListNode head) {
        if (head == null || head.next == null) return head; // 空或单节点
        ListNode newHead = reverseList(head.next); // 先反转后面，返回新头
        head.next.next = head; // 让下一个节点指回自己
        head.next = null;      // 断开自己原来的指向
        return newHead;
    }
}
```

### 小结 / 语法点
- 反转三指针 `prev/cur/next` 是链表最核心的基本功，后面 25 题、234 题都要复用。
- **改指针前先暂存 `next`** 是铁律。

## 234. 回文链表（Palindrome Linked List）· Easy

🔗 https://leetcode.cn/problems/palindrome-linked-list/

### 题目大意
判断链表是否为回文。`1→2→2→1` → true。要求 O(1) 空间最佳。

### 思路
**快慢指针找中点 → 反转后半 → 双指针逐一比对**：
1. 快慢指针，`fast` 走到末尾时 `slow` 在中点。
2. 反转从 `slow` 开始的后半部分。
3. 一个指针从头、一个从反转后的后半头，逐个比较值。
（用"转成数组再双指针"是 O(n) 空间的简单解法，但本题追求 O(1)。）

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public boolean isPalindrome(ListNode head) {
        // 1) 快慢指针找中点：slow 停在后半部分的起点
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }

        // 2) 反转后半部分
        ListNode second = reverse(slow);

        // 3) 前半 head 与反转后的后半 second 逐一比对
        ListNode p1 = head, p2 = second;
        boolean res = true;
        while (p2 != null) {            // 后半较短，以它为准
            if (p1.val != p2.val) { res = false; break; }
            p1 = p1.next;
            p2 = p2.next;
        }
        return res;
    }

    private ListNode reverse(ListNode head) {
        ListNode prev = null, cur = head;
        while (cur != null) {
            ListNode next = cur.next;
            cur.next = prev;
            prev = cur;
            cur = next;
        }
        return prev;
    }
}
```

### 小结 / 语法点
- "**快慢指针找中点 + 反转后半**"是回文链表的标准 O(1) 解法，复用了 206 的反转。
- 比较时以**后半（较短）**为终止条件，奇数长度时中间节点不影响判断。

## 141. 环形链表（Linked List Cycle）· Easy

🔗 https://leetcode.cn/problems/linked-list-cycle/

### 题目大意
判断链表是否有环。

### 思路
**快慢指针（Floyd 判圈）**。慢指针每次 1 步、快指针每次 2 步。若有环，快指针会在环内"套圈"追上慢指针（相遇）；若无环，快指针先到 null。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
public class Solution {
    public boolean hasCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;         // 慢指针走 1 步
            fast = fast.next.next;    // 快指针走 2 步
            if (slow == fast) return true; // 相遇 ⇒ 有环
        }
        return false;                 // 快指针到达 null ⇒ 无环
    }
}
```

### 小结 / 语法点
- 循环条件 `fast != null && fast.next != null`：因为要访问 `fast.next.next`，两个都得非空。
- 快慢指针是判环的标准方法，比哈希表省空间。

## 142. 环形链表 II（Linked List Cycle II）· Medium

🔗 https://leetcode.cn/problems/linked-list-cycle-ii/

### 题目大意
若有环，返回**入环的第一个节点**；否则返回 `null`。

### 思路
**Floyd 判圈 + 数学结论**。设头到入环口距离 `a`，入环口到相遇点 `b`，相遇点回到入环口 `c`。
快指针走的路程是慢指针的 2 倍：`a+b+c+b = 2(a+b)` ⇒ 推出 `a = c`。
所以：快慢相遇后，让一个指针从**头**出发、另一个从**相遇点**出发，每次各走 1 步，它们会在**入环口**相遇。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) {                 // 第一次相遇
                ListNode p = head;              // 一个指针从头出发
                while (p != slow) {             // 另一个从相遇点出发
                    p = p.next;
                    slow = slow.next;
                }
                return p;                       // 相遇处即入环口（a == c）
            }
        }
        return null;                            // 无环
    }
}
```

### 小结 / 语法点
- 记住结论 **`a = c`**：相遇后一个回头、一个续走，在入环口会合。
- 推导不必每次重来，记住"从头和从相遇点同速走，必在环口相遇"即可。

## 21. 合并两个有序链表（Merge Two Sorted Lists）· Easy

🔗 https://leetcode.cn/problems/merge-two-sorted-lists/

### 题目大意
合并两个升序链表，返回合并后的升序链表。

### 思路
**dummy 虚拟头 + 双指针**。比较两表头部，把较小的接到结果尾部并前移，直到某表为空，再把另一表剩余整段接上。

### 复杂度
- 时间 O(m+n)，空间 O(1)。

### 代码
```java
class Solution {
    public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(0); // 虚拟头，简化"第一个节点"的处理
        ListNode cur = dummy;             // 结果链表的尾指针

        while (l1 != null && l2 != null) {
            if (l1.val <= l2.val) {       // 取较小者接上
                cur.next = l1;
                l1 = l1.next;
            } else {
                cur.next = l2;
                l2 = l2.next;
            }
            cur = cur.next;
        }
        // 把还没接完的那一段整体接上（其中一个必为 null）
        cur.next = (l1 != null) ? l1 : l2;

        return dummy.next;                // 跳过虚拟头
    }
}
```

### 小结 / 语法点
- **dummy + 尾指针 cur** 是构造新链表的通用套路。
- 收尾一行 `cur.next = l1 != null ? l1 : l2` 省去逐个接剩余节点。

## 2. 两数相加（Add Two Numbers）· Medium

🔗 https://leetcode.cn/problems/add-two-numbers/

### 题目大意
两个链表逆序存储非负整数（个位在头），逐位相加，返回逆序的和链表。
例：`(2→4→3) + (5→6→4)` = `342+465=807` → `7→0→8`。

### 思路
**模拟竖式加法 + 进位**。同时遍历两表，对应位相加再加进位 `carry`，结果位是 `sum % 10`，新进位是 `sum / 10`。循环条件涵盖"两表未尽 **或** 还有进位"。逆序存储恰好让我们从个位开始算，天然顺手。

### 复杂度
- 时间 O(max(m,n))，空间 O(max(m,n))。

### 代码
```java
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(0);
        ListNode cur = dummy;
        int carry = 0;                       // 进位

        // 只要还有节点没加完，或还有进位，就继续
        while (l1 != null || l2 != null || carry != 0) {
            int sum = carry;                 // 先加上进位
            if (l1 != null) { sum += l1.val; l1 = l1.next; }
            if (l2 != null) { sum += l2.val; l2 = l2.next; }

            carry = sum / 10;                // 新进位
            cur.next = new ListNode(sum % 10); // 当前位
            cur = cur.next;
        }
        return dummy.next;
    }
}
```

### 小结 / 语法点
- 循环条件带上 `carry != 0`，处理最高位进位（如 `5+5=10`）。
- 把"进位"统一在循环内处理，代码简洁，是模拟加法的标准写法。

## 19. 删除链表的倒数第 N 个结点（Remove Nth Node From End）· Medium

🔗 https://leetcode.cn/problems/remove-nth-node-from-end-of-list/

### 题目大意
删除链表的倒数第 `n` 个节点，返回头结点。要求一趟扫描。

### 思路
**快慢指针 + dummy**。让 `fast` 先走 `n` 步，然后 `fast`、`slow` 同速前进，直到 `fast` 到达最后一个节点。此时 `slow` 恰好停在"要删节点的前一个"，`slow.next = slow.next.next` 即删除。
用 dummy 作为 slow 起点，能优雅处理"删除的是头结点"的情况。

### 复杂度
- 时间 O(n)（一趟），空间 O(1)。

### 代码
```java
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        ListNode dummy = new ListNode(0, head); // 防止删头结点的特判
        ListNode fast = dummy, slow = dummy;

        // fast 先走 n 步，制造 n 的间距
        for (int i = 0; i < n; i++) fast = fast.next;

        // 一起走，直到 fast 到最后一个节点；此时 slow 在待删节点的前一个
        while (fast.next != null) {
            fast = fast.next;
            slow = slow.next;
        }

        slow.next = slow.next.next;             // 跳过待删节点
        return dummy.next;
    }
}
```

### 小结 / 语法点
- **固定间距双指针**：让 fast 领先 slow 恰好 n 步，是"找倒数第 k 个"的通用招。
- dummy 让"删头结点"无需特判。

## 24. 两两交换链表中的节点（Swap Nodes in Pairs）· Medium

🔗 https://leetcode.cn/problems/swap-nodes-in-pairs/

### 题目大意
两两相邻节点交换。`1→2→3→4` → `2→1→4→3`。不能只改值。

### 思路
**dummy + 三指针手动改链**。维护交换对的前驱 `prev`。每轮取出 `first`、`second` 两节点，把它们交换：
`first.next = second.next` → `second.next = first` → `prev.next = second`。然后 `prev` 移到 `first`（交换后它是这对的尾），处理下一对。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummy = new ListNode(0, head);
        ListNode prev = dummy;                  // 指向每对的前驱

        // 还剩至少两个节点才能交换
        while (prev.next != null && prev.next.next != null) {
            ListNode first = prev.next;         // 这对的第一个
            ListNode second = first.next;       // 这对的第二个

            // 三步改指针完成交换
            first.next = second.next;           // first 接到这对后面
            second.next = first;                // second 指向 first
            prev.next = second;                 // 前驱接上新的头 second

            prev = first;                       // first 现在是这对的尾，作为下一对前驱
        }
        return dummy.next;
    }
}
```

### 小结 / 语法点
- 三步改链顺序要记牢：**先接 first 的后继，再 second→first，再 prev→second**。
- 画图（箭头）辅助，链表改指针类题一定要在纸上画。

## 25. K 个一组翻转链表（Reverse Nodes in k-Group）· Hard

🔗 https://leetcode.cn/problems/reverse-nodes-in-k-group/

### 题目大意
每 `k` 个节点一组翻转，不足 `k` 个的最后一组保持原样。`1→2→3→4→5, k=2` → `2→1→4→3→5`。

### 思路
**分组 + 组内反转 + 接回**（206 的进阶）。维护 `prevGroup`（上一组的尾，初始 dummy）：
1. 从 `prevGroup` 往后数 k 个，找到本组末尾 `kth`。若不足 k 个（`kth == null`）→ 结束。
2. 记下下一组起点 `nextGroup = kth.next`。
3. 反转本组 `[groupStart, kth]` 这 k 个节点（反转时令初始 `prev = nextGroup`，使本组反转后尾部自动接上下一组）。
4. 接回：`prevGroup.next = kth`（连到新头），`prevGroup = groupStart`（旧头变新尾，作下组前驱）。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        ListNode dummy = new ListNode(0, head);
        ListNode prevGroup = dummy;             // 上一组的尾节点

        while (true) {
            // 1) 从 prevGroup 往后数 k 个，定位本组末尾 kth
            ListNode kth = prevGroup;
            for (int i = 0; i < k && kth != null; i++) kth = kth.next;
            if (kth == null) break;             // 不足 k 个，保持原样，结束

            ListNode groupStart = prevGroup.next; // 本组第一个
            ListNode nextGroup = kth.next;        // 下一组第一个

            // 2) 反转本组：prev 初值设为 nextGroup，让本组反转后自动接上下一组
            ListNode prev = nextGroup, cur = groupStart;
            while (cur != nextGroup) {
                ListNode next = cur.next;
                cur.next = prev;
                prev = cur;
                cur = next;
            }

            // 3) 接回：prevGroup 接新头 kth；groupStart 成为本组新尾，作下组前驱
            prevGroup.next = kth;
            prevGroup = groupStart;
        }
        return dummy.next;
    }
}
```

### 小结 / 语法点
- 难点在"反转后如何无缝接回"，技巧是把组内反转的 `prev` 初值设成 `nextGroup`。
- 先数够 k 个再反转，保证不足 k 的尾组不动。

## 138. 随机链表的复制（Copy List with Random Pointer）· Medium

🔗 https://leetcode.cn/problems/copy-list-with-random-pointer/

### 题目大意
链表每个节点除 `next` 外还有一个 `random` 指针（指向任意节点或 null），深拷贝整个链表。

> 节点定义：`class Node { int val; Node next; Node random; Node(int val){this.val=val;} }`

### 思路
**哈希表映射"原节点 → 拷贝节点"**，两趟遍历：
1. 第一趟：为每个原节点创建对应的拷贝节点，存入 `map`（先不连指针）。
2. 第二趟：借助 map，把每个拷贝节点的 `next` 和 `random` 接到"原节点对应指针所指节点"的拷贝上。
`map.get(null)` 返回 null，天然处理空指针。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public Node copyRandomList(Node head) {
        if (head == null) return null;
        Map<Node, Node> map = new HashMap<>(); // 原节点 -> 拷贝节点

        // 1) 先复制所有节点（只建节点，不接指针）
        Node cur = head;
        while (cur != null) {
            map.put(cur, new Node(cur.val));
            cur = cur.next;
        }

        // 2) 接 next 和 random（用 map 找对应的拷贝节点）
        cur = head;
        while (cur != null) {
            map.get(cur).next = map.get(cur.next);     // null 也能查，返回 null
            map.get(cur).random = map.get(cur.random);
            cur = cur.next;
        }
        return map.get(head);
    }
}
```

### 小结 / 语法点
- 深拷贝带"任意指针"的结构，**哈希表存原→新映射**是最直观解法。
- `map.get(null) == null`，省去对 null 的特判。
- 进阶有 O(1) 空间的"原地交织"解法，先掌握哈希法。

## 148. 排序链表（Sort List）· Medium

🔗 https://leetcode.cn/problems/sort-list/

### 题目大意
在 **O(n log n)** 时间、常数级空间内对链表升序排序。

### 思路
**归并排序**（链表最适合归并，不需随机访问）：
1. 快慢指针找中点，从中间**断开**成两半。
2. 递归排序左右两半。
3. 合并两个有序链表（复用 21 题逻辑）。
（递归栈是 O(log n)，符合要求；纯 O(1) 需自底向上迭代归并，先掌握递归版。）

### 复杂度
- 时间 O(n log n)，空间 O(log n)（递归栈）。

### 代码
```java
class Solution {
    public ListNode sortList(ListNode head) {
        // 递归出口：空或单节点已有序
        if (head == null || head.next == null) return head;

        // 1) 快慢指针找中点。fast 从 head.next 起，保证偶数个时 slow 落在前半末尾
        ListNode slow = head, fast = head.next;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }
        ListNode mid = slow.next;
        slow.next = null;              // 断成两半 [head..slow] 和 [mid..]

        // 2) 递归排序两半
        ListNode left = sortList(head);
        ListNode right = sortList(mid);

        // 3) 合并两个有序链表
        return merge(left, right);
    }

    private ListNode merge(ListNode a, ListNode b) {
        ListNode dummy = new ListNode(0), cur = dummy;
        while (a != null && b != null) {
            if (a.val <= b.val) { cur.next = a; a = a.next; }
            else { cur.next = b; b = b.next; }
            cur = cur.next;
        }
        cur.next = (a != null) ? a : b;
        return dummy.next;
    }
}
```

### 小结 / 语法点
- 链表排序首选**归并**：找中点断链 → 递归 → 合并。
- `fast = head.next` 的小技巧保证两半划分均衡、避免单节点死递归。

## 23. 合并 K 个升序链表（Merge k Sorted Lists）· Hard

🔗 https://leetcode.cn/problems/merge-k-sorted-lists/

### 题目大意
合并 `k` 个升序链表为一个升序链表。

### 思路
**最小堆（优先队列）**。把每个链表的当前头节点放进小根堆（按 `val` 排序），每次弹出最小的接到结果尾部，并把它的 `next`（若有）再入堆。堆顶始终是所有待选节点里最小的。
（另一种是"两两分治合并"，时间同为 O(N log k)。）

### 复杂度
- 时间 O(N log k)，N 为总节点数，k 为链表数。空间 O(k)。

### 代码
```java
class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        // 小根堆，按节点值排序
        PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);

        // 各链表的头节点先入堆
        for (ListNode node : lists) {
            if (node != null) pq.offer(node);
        }

        ListNode dummy = new ListNode(0), cur = dummy;
        while (!pq.isEmpty()) {
            ListNode node = pq.poll();       // 当前最小节点
            cur.next = node;
            cur = cur.next;
            if (node.next != null) pq.offer(node.next); // 它的后继补入堆
        }
        return dummy.next;
    }
}
```

### 小结 / 语法点
- "合并/Top K 个有序序列" → **优先队列**。每次只维护 k 个候选，堆顶取最小。
- 比较器 `(a,b)->a.val-b.val` 让堆按节点值排序。

## 146. LRU 缓存（LRU Cache）· Medium

🔗 https://leetcode.cn/problems/lru-cache/

### 题目大意
设计 LRU（最近最少使用）缓存，`get` 和 `put` 都要 **O(1)**。容量满时淘汰最久未使用的。

### 思路
**哈希表 + 双向链表**（经典组合）：
- **双向链表**按"使用新近度"排序：头部是最近用的，尾部是最久没用的。双向才能 O(1) 删除任意节点。
- **哈希表** `key → 节点`，O(1) 定位节点。
- `get`：命中则把节点移到头部，返回值；未命中返回 -1。
- `put`：已存在则更新值并移到头部；不存在则新建插到头部，若超容则删尾部节点（并从 map 移除）。
- 用 `head`、`tail` 两个**哨兵节点**，让插入/删除无需判空。

### 复杂度
- `get`、`put` 均 O(1)。空间 O(capacity)。

### 代码
```java
class LRUCache {
    // 双向链表节点
    class Node {
        int key, value;
        Node prev, next;
        Node(int k, int v) { key = k; value = v; }
    }

    private Map<Integer, Node> map = new HashMap<>();
    private int capacity;
    private Node head, tail;   // 哨兵：head.next 最近使用，tail.prev 最久未使用

    public LRUCache(int capacity) {
        this.capacity = capacity;
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head.next = tail;      // 初始空链表：head <-> tail
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        moveToHead(node);      // 访问即刷新到最近使用
        return node.value;
    }

    public void put(int key, int value) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.value = value;        // 更新值
            moveToHead(node);
        } else {
            if (map.size() == capacity) {     // 满了先淘汰最久未使用（尾部）
                Node lru = tail.prev;
                remove(lru);
                map.remove(lru.key);
            }
            Node node = new Node(key, value);
            map.put(key, node);
            addToHead(node);                  // 新节点插到头部
        }
    }

    // —— 双向链表基本操作 ——
    private void remove(Node node) {          // 摘除节点
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }
    private void addToHead(Node node) {       // 插到头部（head 之后）
        node.next = head.next;
        node.prev = head;
        head.next.prev = node;
        head.next = node;
    }
    private void moveToHead(Node node) {      // = 摘除 + 插到头部
        remove(node);
        addToHead(node);
    }
}
```

### 小结 / 语法点
- **LRU = HashMap（O(1) 查）+ 双向链表（O(1) 调整顺序）**，是必背的设计题。
- 用 `head`/`tail` 哨兵节点，省去插入删除时的边界判空。
- 把 `remove`/`addToHead`/`moveToHead` 拆成小方法，逻辑清爽不易错。
- （Java 还可用 `LinkedHashMap` 重写 `removeEldestEntry` 几行实现，但面试要求手写双向链表版。）

## 70. 爬楼梯（Climbing Stairs）· Easy

🔗 https://leetcode.cn/problems/climbing-stairs/

### 题目大意
每次爬 1 或 2 个台阶，爬到第 `n` 阶有多少种方法。

### 思路
到第 i 阶，要么从 i-1 阶爬 1 步、要么从 i-2 阶爬 2 步，所以 `dp[i] = dp[i-1] + dp[i-2]`（斐波那契）。只依赖前两个状态，用两个滚动变量即可 O(1) 空间。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int climbStairs(int n) {
        if (n <= 2) return n;          // 1阶→1种，2阶→2种
        int a = 1, b = 2;              // a = dp[i-2], b = dp[i-1]
        for (int i = 3; i <= n; i++) {
            int c = a + b;             // dp[i] = dp[i-1] + dp[i-2]
            a = b;
            b = c;
        }
        return b;
    }
}
```

### 小结 / 语法点
- 最经典的"线性 DP + 滚动变量"，转移就是斐波那契。
- 打家劫舍、变态跳台阶都是它的变体。

## 118. 杨辉三角（Pascal's Triangle）· Easy

🔗 https://leetcode.cn/problems/pascals-triangle/

### 题目大意
返回前 `numRows` 行杨辉三角。每个数 = 上方两数之和，两端为 1。

### 思路
**直接递推**。第 i 行第 j 个：边界（`j==0` 或 `j==i`）为 1，否则等于上一行的 `[j-1] + [j]`。

### 复杂度
- 时间 O(n²)，空间 O(n²)（输出本身）。

### 代码
```java
class Solution {
    public List<List<Integer>> generate(int numRows) {
        List<List<Integer>> res = new ArrayList<>();
        for (int i = 0; i < numRows; i++) {
            List<Integer> row = new ArrayList<>();
            for (int j = 0; j <= i; j++) {
                if (j == 0 || j == i) {
                    row.add(1);                 // 两端为 1
                } else {
                    // 上一行的相邻两数之和
                    row.add(res.get(i - 1).get(j - 1) + res.get(i - 1).get(j));
                }
            }
            res.add(row);
        }
        return res;
    }
}
```

### 小结 / 语法点
- 转移方程直白：`row[j] = 上一行[j-1] + 上一行[j]`。
- 用 `res.get(i-1)` 取上一行，注意边界 j==0 / j==i。

## 198. 打家劫舍（House Robber）· Medium

🔗 https://leetcode.cn/problems/house-robber/

### 题目大意
不能偷相邻的两家，求能偷到的最大金额。

### 思路
对第 i 家，**偷**（拿 `dp[i-2] + nums[i]`，跳过相邻）或**不偷**（拿 `dp[i-1]`），取较大：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。同样只依赖前两个状态，滚动变量优化。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int rob(int[] nums) {
        int prev = 0, cur = 0; // prev = dp[i-2], cur = dp[i-1]
        for (int x : nums) {
            int next = Math.max(cur, prev + x); // 不偷 vs 偷当前
            prev = cur;
            cur = next;
        }
        return cur;
    }
}
```

### 小结 / 语法点
- 转移：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`，"不偷"或"隔一家偷"。
- 打家劫舍 II（环形）、III（树形）是延伸，套路一致。

## 279. 完全平方数（Perfect Squares）· Medium

🔗 https://leetcode.cn/problems/perfect-squares/

### 题目大意
求若干个**完全平方数**（1,4,9,16…）相加等于 `n` 所需的**最少个数**。

### 思路
**完全背包**。`dp[i]` = 凑出 i 所需的最少平方数个数。枚举平方数 `j*j ≤ i`，`dp[i] = min(dp[i], dp[i - j*j] + 1)`。初始 `dp[0]=0`，其余设为很大。

### 复杂度
- 时间 O(n√n)，空间 O(n)。

### 代码
```java
class Solution {
    public int numSquares(int n) {
        int[] dp = new int[n + 1];
        Arrays.fill(dp, Integer.MAX_VALUE);
        dp[0] = 0;                         // 凑 0 需要 0 个
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j * j <= i; j++) {   // 枚举用哪个平方数
                dp[i] = Math.min(dp[i], dp[i - j * j] + 1);
            }
        }
        return dp[n];
    }
}
```

### 小结 / 语法点
- "凑出某数的最少个数" → 背包型 DP，`dp[i] = min(dp[i - 物品] + 1)`。
- 和零钱兑换是同一模型，物品换成"平方数"。

## 322. 零钱兑换（Coin Change）· Medium

🔗 https://leetcode.cn/problems/coin-change/

### 题目大意
用给定面额硬币凑出 `amount`，求最少硬币数；凑不出返回 -1。每种硬币无限用。

### 思路
**完全背包**。`dp[i]` = 凑出金额 i 的最少硬币数。对每个金额 i，枚举每种硬币 `coin ≤ i`：`dp[i] = min(dp[i], dp[i - coin] + 1)`。用 `amount + 1` 当"无穷大"标记（因为最多 amount 个 1 元硬币）。最后若 `dp[amount]` 仍是无穷，返回 -1。

### 复杂度
- 时间 O(amount × 硬币种数)，空间 O(amount)。

### 代码
```java
class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1);       // amount+1 当“无穷大”（不可能达到的值）
        dp[0] = 0;                         // 凑 0 需要 0 枚
        for (int i = 1; i <= amount; i++) {
            for (int coin : coins) {
                if (coin <= i) {
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        return dp[amount] > amount ? -1 : dp[amount]; // 没被更新过 → 凑不出
    }
}
```

### 小结 / 语法点
- 经典完全背包"求最少个数"。用 `amount+1` 当无穷大避免溢出，又方便判断"是否可达"。
- 与 279 是同一套路（物品分别是硬币 / 平方数）。

## 139. 单词拆分（Word Break）· Medium

🔗 https://leetcode.cn/problems/word-break/

### 题目大意
判断字符串 `s` 能否被拆分成字典 `wordDict` 中单词的组合（单词可重复用）。

### 思路
**序列 DP**。`dp[i]` = `s` 的前 i 个字符能否被拆分。`dp[0]=true`（空串）。对每个 i，枚举切点 j：若 `dp[j]` 为真**且** `s[j..i)` 在字典里，则 `dp[i]=true`。字典转成 `HashSet` 加速查询。

### 复杂度
- 时间 O(n²)（再乘子串比较），空间 O(n)。

### 代码
```java
class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        Set<String> set = new HashSet<>(wordDict);  // 查询 O(1)
        boolean[] dp = new boolean[s.length() + 1];
        dp[0] = true;                                // 空串可拆
        for (int i = 1; i <= s.length(); i++) {
            for (int j = 0; j < i; j++) {
                // 前 j 个可拆，且 [j, i) 这段是字典里的单词
                if (dp[j] && set.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break;                           // 找到一种拆法即可
                }
            }
        }
        return dp[s.length()];
    }
}
```

### 小结 / 语法点
- `dp[i]` 表示"前缀能否拆分"，转移枚举最后一个单词的起点 j。
- 字典用 `HashSet`；`substring(j, i)` 取 `[j, i)`。

## 300. 最长递增子序列（Longest Increasing Subsequence）· Medium

🔗 https://leetcode.cn/problems/longest-increasing-subsequence/

### 题目大意
求最长**严格递增子序列**的长度（子序列不要求连续）。
例：`[10,9,2,5,3,7,101,18]` → `4`（`2,3,7,101`）。

### 思路
**方法一 O(n²) DP**：`dp[i]` = 以 `nums[i]` 结尾的最长递增子序列长度。对每个 i，看它前面所有 `nums[j] < nums[i]` 的 j，`dp[i] = max(dp[j]) + 1`。
**方法二 O(n log n) 贪心+二分**：维护数组 `tails`，`tails[k]` = 长度为 k+1 的递增子序列的**最小可能结尾**。对每个数二分找它能接的位置：能延长就追加，否则替换。`tails` 的长度就是答案。

### 复杂度
- 方法一 O(n²)；方法二 O(n log n)。空间 O(n)。

### 代码（方法一，先掌握）
```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        int[] dp = new int[n];
        Arrays.fill(dp, 1);            // 每个数本身长度至少为 1
        int max = 1;
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i])         // 可以接在 nums[j] 后面
                    dp[i] = Math.max(dp[i], dp[j] + 1);
            }
            max = Math.max(max, dp[i]);
        }
        return max;
    }
}
```

### 代码（方法二，O(n log n)）
```java
class Solution {
    public int lengthOfLIS(int[] nums) {
        // tails[k]：长度为 k+1 的递增子序列的最小结尾
        List<Integer> tails = new ArrayList<>();
        for (int x : nums) {
            // 二分找第一个 >= x 的位置（lower_bound）
            int l = 0, r = tails.size();
            while (l < r) {
                int mid = l + (r - l) / 2;
                if (tails.get(mid) < x) l = mid + 1;
                else r = mid;
            }
            if (l == tails.size()) tails.add(x);  // x 比所有结尾都大 → 延长
            else tails.set(l, x);                 // 否则替换，让结尾更小
        }
        return tails.size();
    }
}
```

### 小结 / 语法点
- `dp[i]` 定义为"**以 i 结尾**"是 LIS 的关键（不能定义成"前 i 个的 LIS"，否则无法转移）。
- O(n log n) 解法的 `tails` 不是真实子序列，但长度正确——"贪心让结尾尽量小"。

## 152. 乘积最大子数组（Maximum Product Subarray）· Medium

🔗 https://leetcode.cn/problems/maximum-product-subarray/

### 题目大意
找出**乘积最大**的连续子数组，返回该乘积。数组含负数和 0。
例：`[2,3,-2,4]` → `6`（`[2,3]`）。

### 思路
与最大子数组和不同：因为**负负得正**，当前最大值可能来自"前一个最小值 × 当前负数"。所以要**同时维护以当前元素结尾的最大乘积 `max` 和最小乘积 `min`**。
遇到负数时，最大和最小会互换角色（先交换再更新）。每步：`max = max(x, max*x)`，`min = min(x, min*x)`，更新全局答案。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int maxProduct(int[] nums) {
        int max = nums[0], min = nums[0], res = nums[0];
        for (int i = 1; i < nums.length; i++) {
            int x = nums[i];
            if (x < 0) {                       // 负数会让最大/最小互换
                int t = max; max = min; min = t;
            }
            max = Math.max(x, max * x);        // 以 x 结尾的最大乘积
            min = Math.min(x, min * x);        // 以 x 结尾的最小乘积
            res = Math.max(res, max);
        }
        return res;
    }
}
```

### 小结 / 语法点
- 关键：**同时维护最大和最小**，因为负数会翻转大小关系。
- 遇负数先 `swap(max, min)` 再更新，是本题精髓。

## 416. 分割等和子集（Partition Equal Subset Sum）· Medium

🔗 https://leetcode.cn/problems/partition-equal-subset-sum/

### 题目大意
判断数组能否分成两个子集，使两子集元素和相等。

### 思路
等价于"**能否从数组中选出若干个数，和恰好为 `总和/2`**"——经典 **0/1 背包**（每个数选或不选）。
- 若总和为奇数，直接 false。
- `dp[j]` = 能否凑出和 j。`dp[0]=true`。对每个数 x，**倒序**更新 `dp[j] = dp[j] || dp[j-x]`（倒序保证每个数只用一次）。

### 复杂度
- 时间 O(n × target)，空间 O(target)。

### 代码
```java
class Solution {
    public boolean canPartition(int[] nums) {
        int sum = 0;
        for (int x : nums) sum += x;
        if (sum % 2 != 0) return false;        // 奇数和无法平分
        int target = sum / 2;

        boolean[] dp = new boolean[target + 1];
        dp[0] = true;                          // 和为 0 总能达成（什么都不选）
        for (int x : nums) {
            for (int j = target; j >= x; j--) {   // 倒序！保证 x 只被用一次（0/1 背包）
                dp[j] = dp[j] || dp[j - x];
            }
        }
        return dp[target];
    }
}
```

### 小结 / 语法点
- "**能否选出子集凑出某和**" → 0/1 背包布尔 DP。
- **一维背包必须倒序遍历容量**，否则会重复使用同一元素（变成完全背包）。

## 32. 最长有效括号（Longest Valid Parentheses）· Hard

🔗 https://leetcode.cn/problems/longest-valid-parentheses/

### 题目大意
找出最长的**有效（格式正确且连续）**括号子串的长度。
例：`")()())"` → `4`（`"()()"`）。

### 思路
**栈 + 下标**（比 DP 更直观）。栈底始终放一个"最后一个未匹配右括号的下标"作为基准（初始放 -1）。
- 遇 `(`：把下标压栈。
- 遇 `)`：先弹栈。若弹完栈空，说明当前 `)` 没人配对，把它的下标压栈当新基准；否则当前有效长度 = `i - 栈顶下标`，更新最大值。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public int longestValidParentheses(String s) {
        int max = 0;
        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(-1);                        // 基准：最后一个未匹配位置的“前一个”
        for (int i = 0; i < s.length(); i++) {
            if (s.charAt(i) == '(') {
                stack.push(i);                 // 左括号下标入栈
            } else {
                stack.pop();                   // 尝试匹配一个左括号
                if (stack.isEmpty()) {
                    stack.push(i);             // 没匹配上 → 当前右括号成为新基准
                } else {
                    max = Math.max(max, i - stack.peek()); // 有效长度 = 当前下标 - 基准
                }
            }
        }
        return max;
    }
}
```

### 小结 / 语法点
- 栈里存**下标**，栈底维护"基准位置"，有效长度 = `i - 栈顶`。
- 也可用 DP（`dp[i]` = 以 i 结尾的最长有效长度）或左右双向计数器，三种都值得了解。

## 560. 和为 K 的子数组（Subarray Sum Equals K）· Medium

🔗 https://leetcode.cn/problems/subarray-sum-equals-k/

### 题目大意
求数组中**和等于 k** 的**连续子数组**的个数。注意数组可能含**负数**，所以不能用滑动窗口（窗口和不单调）。
例：`nums=[1,1,1], k=2` → `3`（两个长度为 2 的子数组 + 1 个... 实际是 `[1,1]`×2 = 2... 修正见下）。

> 订正示例：`nums=[1,1,1], k=2` → 答案 `2`（下标 `[0,1]` 和 `[1,2]`）。

### 思路
**前缀和 + 哈希**。设 `pre[i]` 为前 i 个元素之和，则子数组 `(j, i]` 的和 = `pre[i] - pre[j]`。
我们要的是 `pre[i] - pre[j] == k`，即 `pre[j] == pre[i] - k`。

于是转化成"**两数之和**"思想：遍历到 i 时，问"前面有多少个前缀和等于 `pre[i] - k`？"——每一个这样的 j 都对应一个和为 k 的子数组。用哈希表记录"**每个前缀和出现的次数**"即可 O(1) 查询。

关键初始化：`cnt.put(0, 1)`，表示"前缀和为 0"出现了 1 次（代表空前缀），这样当 `pre[i]` 本身就等于 k 时也能正确计入。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public int subarraySum(int[] nums, int k) {
        // key = 某个前缀和，value = 该前缀和出现的次数
        Map<Integer, Integer> cnt = new HashMap<>();
        cnt.put(0, 1);             // 空前缀：和为 0 出现 1 次（关键初始化）

        int pre = 0;               // 当前位置的前缀和
        int ans = 0;

        for (int x : nums) {
            pre += x;              // 累加得到当前前缀和

            // 需要的历史前缀和是 pre-k，它出现过几次，就有几个以当前位置结尾、和为 k 的子数组
            ans += cnt.getOrDefault(pre - k, 0);

            // 把当前前缀和登记进表（供后面的位置查询）
            cnt.merge(pre, 1, Integer::sum);
        }
        return ans;
    }
}
```

### 小结 / 语法点
- **含负数 ⇒ 不能滑窗 ⇒ 想前缀和**。这是重要的判断信号。
- 转化为"两数之和"：把 `pre - k` 当成要找的"另一半"。
- `cnt.put(0, 1)` 这个初始化极易漏，务必记住。
- `cnt.merge(pre, 1, Integer::sum)` 等价于"出现次数 +1"。

## 239. 滑动窗口最大值（Sliding Window Maximum）· Hard

🔗 https://leetcode.cn/problems/sliding-window-maximum/

### 题目大意
大小为 `k` 的窗口从左到右滑过数组，返回每个窗口的**最大值**。
例：`nums=[1,3,-1,-3,5,3,6,7], k=3` → `[3,3,5,5,6,7]`。

### 思路
**单调队列**（双端队列存**下标**，对应的值从队头到队尾**单调递减**）。
- 队头始终是当前窗口的最大值下标。
- 新元素进来时，把队尾所有**比它小**的下标弹掉（它们不可能再成为最大值，因为有更靠右且更大的它在），保持队列递减。
- 队头若已经滑出窗口左边界（下标 ≤ i - k），从队头弹出。
- 窗口形成后（i ≥ k-1），队头对应的值就是当前窗口最大值。

为什么用单调队列而不是堆？堆是 O(n log k)，单调队列 O(n)——每个下标最多入队、出队各一次。

### 复杂度
- 时间 O(n)，空间 O(k)。

### 代码
```java
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] res = new int[n - k + 1];        // 共 n-k+1 个窗口
        // 双端队列存下标，保证 nums[队列] 从队头到队尾单调递减
        Deque<Integer> dq = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            // 1) 维持单调递减：弹掉队尾所有 <= 当前值的下标（它们被当前值“盖过”）
            while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) {
                dq.pollLast();
            }
            dq.offerLast(i);                   // 当前下标入队尾

            // 2) 队头若滑出窗口（下标超出左边界 i-k），弹出队头
            if (dq.peekFirst() <= i - k) {
                dq.pollFirst();
            }

            // 3) 窗口形成后（下标 i 到达 k-1），队头即最大值
            if (i >= k - 1) {
                res[i - k + 1] = nums[dq.peekFirst()];
            }
        }
        return res;
    }
}
```

### 小结 / 语法点
- 单调队列**存下标**而非值，因为要靠下标判断是否滑出窗口。
- 三步顺序固定：先维护单调性入队 → 再弹过期队头 → 再取答案。
- `Deque` 两头都能操作：`offerLast/pollLast` 管队尾，`peekFirst/pollFirst` 管队头。

## 76. 最小覆盖子串（Minimum Window Substring）· Hard

🔗 https://leetcode.cn/problems/minimum-window-substring/

### 题目大意
在 `s` 中找出**最短**的子串，使其包含 `t` 的**所有字符**（含重复次数）。不存在返回 `""`。
例：`s="ADOBECODEBANC", t="ABC"` → `"BANC"`。

### 思路
**带计数的不定长滑动窗口（求最短）**：
- `need[c]`：`t` 中字符 c 需要的数量；`required`：`t` 中不同字符的种数。
- `window[c]`：当前窗口中字符 c 的数量；`formed`：当前已"满足数量要求"的字符种数。
- 右指针扩张，把字符纳入窗口；当某字符在窗口里的数量**恰好达到** need 时，`formed++`。
- 当 `formed == required`（所有字符都齐了），窗口合法 → 尝试**收缩左指针**记录更短的答案；收缩时若某字符数量掉到 need 以下，`formed--`，停止收缩。

"求最短"和"求最长"的区别：求最短在 **while(满足)** 循环内部更新答案，并持续收缩压榨长度。

### 复杂度
- 时间 O(|s| + |t|)，每个字符最多进出窗口各一次。
- 空间 O(字符集) = O(128)。

### 代码
```java
class Solution {
    public String minWindow(String s, String t) {
        int[] need = new int[128];     // t 中每个字符需要的数量（ASCII 直接当下标）
        int required = 0;              // t 中不同字符的种数
        for (char c : t.toCharArray()) {
            if (need[c]++ == 0) required++; // 第一次出现该字符，种类数 +1
        }

        int[] window = new int[128];   // 当前窗口各字符数量
        int formed = 0;                // 已“数量达标”的字符种数
        int left = 0;
        int minLen = Integer.MAX_VALUE, start = 0; // 记录最优窗口的长度与起点

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            window[c]++;                          // 右端字符进窗口
            if (window[c] == need[c]) formed++;   // 该字符数量刚好达标

            // 所有字符都齐了 → 窗口合法，尝试收缩左边界以求更短
            while (formed == required) {
                // 更新最优解
                if (right - left + 1 < minLen) {
                    minLen = right - left + 1;
                    start = left;
                }
                // 移出左端字符
                char lc = s.charAt(left);
                window[lc]--;
                if (window[lc] < need[lc]) formed--; // 该字符不再达标，停止收缩
                left++;
            }
        }

        return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
    }
}
```

### 小结 / 语法点
- `need`/`window` 用 `int[128]` 覆盖大小写字母，比 HashMap 更快。
- `required` 与 `formed` 这对计数器是"判断窗口是否覆盖 t"的高效手段，避免每次全量比对。
- **求最短在 while 内更新答案**；对照 3 题"求最长在 while 外更新答案"，体会两种滑窗的差别。
- 用 `start + minLen` 配合 `substring` 取出结果，避免存整个子串。

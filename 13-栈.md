## 20. 有效的括号（Valid Parentheses）· Easy

🔗 https://leetcode.cn/problems/valid-parentheses/

### 题目大意
判断只含 `()[]{}` 的字符串括号是否有效（正确闭合、正确嵌套）。

### 思路
**栈**。遇左括号，把它**期望的右括号**压栈；遇右括号，检查栈顶是否正好是它——不匹配或栈空则非法。遍历完栈空才有效。
（压"期望的右括号"是个小技巧，省去匹配映射表的判断。）

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c == '(') stack.push(')');           // 压入“期望的右括号”
            else if (c == '[') stack.push(']');
            else if (c == '{') stack.push('}');
            else {
                // 遇右括号：栈空 或 栈顶不是它 → 非法
                if (stack.isEmpty() || stack.pop() != c) return false;
            }
        }
        return stack.isEmpty();                      // 全部配对完，栈应为空
    }
}
```

### 小结 / 语法点
- 括号匹配 = 栈的经典应用："最近的左括号最先被闭合"正好是 LIFO。
- 压入"期望的右括号"让右括号判断变成一次相等比较。

## 155. 最小栈（Min Stack）· Medium

🔗 https://leetcode.cn/problems/min-stack/

### 题目大意
设计栈，支持 `push/pop/top`，并能在 **O(1)** 时间获取栈中最小值 `getMin`。

### 思路
**辅助栈同步记录最小值**。主栈正常存数据；辅助栈 `minStack` 与主栈同步入出，栈顶始终是"当前主栈的最小值"。push 时，新最小值 = min(新元素, 之前的最小)，压入辅助栈。这样任何时刻 `minStack.peek()` 都是 O(1) 的当前最小。

### 复杂度
- 所有操作 O(1)，空间 O(n)。

### 代码
```java
class MinStack {
    private Deque<Integer> stack = new ArrayDeque<>();    // 主栈
    private Deque<Integer> minStack = new ArrayDeque<>(); // 辅助栈，栈顶=当前最小

    public MinStack() {}

    public void push(int val) {
        stack.push(val);
        // 辅助栈同步压入“到目前为止的最小值”
        if (minStack.isEmpty() || val <= minStack.peek()) minStack.push(val);
        else minStack.push(minStack.peek());
    }

    public void pop() {
        stack.pop();
        minStack.pop();         // 同步弹出，保持对齐
    }

    public int top() {
        return stack.peek();
    }

    public int getMin() {
        return minStack.peek(); // O(1) 取最小
    }
}
```

### 小结 / 语法点
- **辅助栈**是"O(1) 维护某统计量"的常用手法。
- 两栈同步 push/pop 保证下标对齐，`getMin` 直接读辅助栈顶。

## 394. 字符串解码（Decode String）· Medium

🔗 https://leetcode.cn/problems/decode-string/

### 题目大意
解码形如 `k[encoded]` 的字符串，`k` 表示方括号内内容重复次数（可嵌套）。
例：`"3[a2[c]]"` → `"accaccacc"`。

### 思路
**双栈处理嵌套**。一个栈存"重复次数"，一个栈存"进入括号前已拼好的字符串"。
- 数字：累积成完整的 `k`（可能多位）。
- `[`：把当前 `num` 和 `cur` 分别压栈，重置它们，开始记录括号内内容。
- `]`：弹出重复次数 `k` 和括号外的前缀 `prev`，令 `cur = prev + cur重复k次`。
- 字母：追加到 `cur`。

### 复杂度
- 时间 O(输出长度)，空间 O(嵌套深度)。

### 代码
```java
class Solution {
    public String decodeString(String s) {
        Deque<Integer> numStack = new ArrayDeque<>();         // 重复次数栈
        Deque<StringBuilder> strStack = new ArrayDeque<>();   // 括号外前缀栈
        StringBuilder cur = new StringBuilder();              // 当前层正在拼的字符串
        int num = 0;                                          // 当前重复次数

        for (char c : s.toCharArray()) {
            if (Character.isDigit(c)) {
                num = num * 10 + (c - '0');                   // 多位数字累积
            } else if (c == '[') {
                numStack.push(num);                          // 保存次数
                strStack.push(cur);                          // 保存括号外已拼内容
                num = 0;
                cur = new StringBuilder();                   // 进入新一层
            } else if (c == ']') {
                int repeat = numStack.pop();
                StringBuilder temp = cur;                    // 括号内内容
                cur = strStack.pop();                        // 恢复括号外前缀
                for (int i = 0; i < repeat; i++) cur.append(temp); // 重复拼接
            } else {
                cur.append(c);                               // 普通字母
            }
        }
        return cur.toString();
    }
}
```

### 小结 / 语法点
- **嵌套结构 → 栈**。数字栈 + 字符串栈分别保存"重复几次"和"括号外的上文"。
- 遇 `]` 时"出栈 + 重复拼接"，正好还原一层嵌套。

## 739. 每日温度（Daily Temperatures）· Medium

🔗 https://leetcode.cn/problems/daily-temperatures/

### 题目大意
给每日温度数组，返回每天"还要等几天才会更暖和"；以后都不会更暖则填 0。
例：`[73,74,75,71,69,72,76,73]` → `[1,1,4,2,1,1,0,0]`。

### 思路
**单调栈（递减）**，栈里存**下标**。遍历，当前温度比栈顶那天高时，栈顶那天的"等待天数"就确定了（= 当前下标 - 栈顶下标），弹出并填结果。直到栈顶比当前高或栈空，再把当前下标压栈。

### 复杂度
- 时间 O(n)（每个下标进出栈各一次），空间 O(n)。

### 代码
```java
class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] res = new int[n];                  // 默认全 0（之后没有更暖的）
        Deque<Integer> stack = new ArrayDeque<>(); // 单调递减栈，存下标

        for (int i = 0; i < n; i++) {
            // 当前温度更高 → 结算栈顶那些天
            while (!stack.isEmpty() && temperatures[i] > temperatures[stack.peek()]) {
                int idx = stack.pop();
                res[idx] = i - idx;              // 等待天数 = 下标差
            }
            stack.push(i);                       // 当前下标入栈，等待将来更高温
        }
        return res;
    }
}
```

### 小结 / 语法点
- "**下一个更大元素**"类问题 = 单调栈。栈存下标方便算距离。
- 维护**递减**栈：来更大值时把小的弹出结算。

## 84. 柱状图中最大的矩形（Largest Rectangle in Histogram）· Hard

🔗 https://leetcode.cn/problems/largest-rectangle-in-histogram/

### 题目大意
给柱状图各柱高度，求能勾勒出的最大矩形面积。
例：`[2,1,5,6,2,3]` → `10`（高 5、6 那两根，宽 2，面积 10）。

### 思路
**单调递增栈**。对每根柱子，要找它**左右两侧第一个比它矮**的柱子，宽度就是这两个边界之间的距离，面积 = 高 × 宽。
用单调递增栈：当遇到比栈顶矮的柱子时，栈顶柱子的右边界就确定了（当前柱），左边界是栈里的前一个，结算它的面积。
**两端加哨兵 0**：左哨兵保证栈不空（左边界好算），右哨兵保证遍历结束时把栈里所有柱子都结算掉。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public int largestRectangleArea(int[] heights) {
        int n = heights.length;
        int[] h = new int[n + 2];                // 两端各加一个高度为 0 的哨兵
        for (int i = 0; i < n; i++) h[i + 1] = heights[i];

        Deque<Integer> stack = new ArrayDeque<>(); // 单调递增栈，存下标
        int max = 0;
        for (int i = 0; i < h.length; i++) {
            // 遇到更矮的柱子 → 结算栈顶柱子能撑的最大矩形
            while (!stack.isEmpty() && h[i] < h[stack.peek()]) {
                int height = h[stack.pop()];     // 被结算柱子的高
                int width = i - stack.peek() - 1;// 宽 = 右边界 i 到左边界(新栈顶)之间
                max = Math.max(max, height * width);
            }
            stack.push(i);
        }
        return max;
    }
}
```

### 小结 / 语法点
- 关键：求每根柱"左右第一个更矮"用单调栈；宽度 = `右边界 - 左边界 - 1`。
- **左右哨兵 0** 是简化边界的经典技巧（左哨兵让栈始终非空，右哨兵清空栈）。
- 接雨水（42）是它的"对偶"题，都靠单调栈/双指针处理"边界更矮/更高"。

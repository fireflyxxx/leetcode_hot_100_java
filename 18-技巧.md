## 136. 只出现一次的数字（Single Number）· Easy

🔗 https://leetcode.cn/problems/single-number/

### 题目大意
数组中**只有一个**元素出现一次，其余都出现两次，找出那个数。要求线性时间、O(1) 空间。

### 思路
**异或（XOR）**。利用两条性质：`x ^ x = 0`（相同抵消）、`x ^ 0 = x`。把所有数异或起来，成对的互相抵消为 0，最后剩下的就是那个只出现一次的数。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int singleNumber(int[] nums) {
        int res = 0;
        for (int x : nums) res ^= x; // 全部异或：成对抵消，剩单独的
        return res;
    }
}
```

### 小结 / 语法点
- 异或三定律：`x^x=0`、`x^0=x`、满足交换律结合律。
- "找成对中的单个" / "找缺失的数" 经常用异或。

## 169. 多数元素（Majority Element）· Easy

🔗 https://leetcode.cn/problems/majority-element/

### 题目大意
找出出现次数 **> ⌊n/2⌋** 的元素（保证存在）。要求 O(n) 时间、O(1) 空间。

### 思路
**Boyer-Moore 摩尔投票**。维护一个候选 `candidate` 和计数 `count`：
- `count == 0` 时，把当前数设为新候选。
- 当前数等于候选则 `count++`，否则 `count--`。
因为多数元素超过半数，不同元素相互抵消后，最终剩下的候选一定是多数元素。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int majorityElement(int[] nums) {
        int candidate = nums[0], count = 0;
        for (int x : nums) {
            if (count == 0) candidate = x;          // 票数归零，换候选
            count += (x == candidate) ? 1 : -1;     // 同票 +1，异票 -1
        }
        return candidate;
    }
}
```

### 小结 / 语法点
- 摩尔投票核心："**不同的两个数相互抵消，多数派必然笑到最后**"。
- 因为题目保证多数元素存在，无需第二趟验证。

## 75. 颜色分类（Sort Colors）· Medium

🔗 https://leetcode.cn/problems/sort-colors/

### 题目大意
数组只含 0、1、2，原地排序使相同颜色相邻、按 0→1→2 排列（荷兰国旗问题）。要求一趟扫描。

### 思路
**三指针（荷兰国旗）**。`p0` 指向"下一个 0 应放的位置"，`p2` 指向"下一个 2 应放的位置"，`i` 是当前扫描指针：
- `nums[i] == 0`：和 `p0` 交换，`p0++`、`i++`。
- `nums[i] == 2`：和 `p2` 交换，`p2--`；**`i` 不动**（换过来的数还没检查）。
- `nums[i] == 1`：`i++`。
当 `i > p2` 时结束。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public void sortColors(int[] nums) {
        int p0 = 0, p2 = nums.length - 1, i = 0;
        while (i <= p2) {
            if (nums[i] == 0) {
                swap(nums, i, p0);   // 0 换到前面
                p0++;
                i++;                 // p0 之前都已处理，i 可以前进
            } else if (nums[i] == 2) {
                swap(nums, i, p2);   // 2 换到后面
                p2--;
                // i 不前进！换过来的值还没检查
            } else {
                i++;                 // 1 留在中间
            }
        }
    }
    private void swap(int[] nums, int a, int b) {
        int t = nums[a]; nums[a] = nums[b]; nums[b] = t;
    }
}
```

### 小结 / 语法点
- 三指针把数组分成 `[0...p0) 全0 | [p0...i) 全1 | (p2...] 全2` 三段。
- **换 2 之后 `i` 不动**是最易错的点：换过来的元素还需判断。

## 31. 下一个排列（Next Permutation）· Medium

🔗 https://leetcode.cn/problems/next-permutation/

### 题目大意
把数组重排成**字典序中下一个更大的排列**；若已是最大（降序），则变成最小（升序）。要求原地、O(1) 空间。
例：`[1,2,3]` → `[1,3,2]`；`[3,2,1]` → `[1,2,3]`。

### 思路
**三步法**（找规律）：
1. 从右往左，找第一个**升序对** `nums[i] < nums[i+1]`，记下 `i`（这是需要变大的"低位"）。
2. 若找到了 i：再从右往左找第一个**大于 `nums[i]`** 的数 `nums[j]`，交换 `i`、`j`（让 i 处变成稍大的数）。
3. 反转 `i+1` 到末尾（此时这段是降序，反转成升序 = 最小），使后缀最小。
若第 1 步没找到（整个降序），i = -1，直接反转整个数组变升序。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public void nextPermutation(int[] nums) {
        int n = nums.length;
        int i = n - 2;
        // 1) 从右找第一个升序对 nums[i] < nums[i+1]
        while (i >= 0 && nums[i] >= nums[i + 1]) i--;

        if (i >= 0) {
            // 2) 从右找第一个比 nums[i] 大的数，交换
            int j = n - 1;
            while (nums[j] <= nums[i]) j--;
            swap(nums, i, j);
        }
        // 3) 反转 i+1 到末尾（使后缀升序 = 最小）
        reverse(nums, i + 1, n - 1);
    }
    private void swap(int[] nums, int a, int b) {
        int t = nums[a]; nums[a] = nums[b]; nums[b] = t;
    }
    private void reverse(int[] nums, int l, int r) {
        while (l < r) { swap(nums, l, r); l++; r--; }
    }
}
```

### 小结 / 语法点
- 三步口诀：**右找升序对 → 右找更大数交换 → 反转后缀**。
- 整体降序时 i = -1，第 3 步 `reverse(nums, 0, n-1)` 直接得到最小排列。

## 287. 寻找重复数（Find the Duplicate Number）· Medium

🔗 https://leetcode.cn/problems/find-the-duplicate-number/

### 题目大意
长度为 `n+1` 的数组，元素都在 `[1, n]`，**只有一个重复的数**（可能重复多次），找出它。要求**不修改数组**、O(1) 空间。

### 思路
**快慢指针（Floyd 判圈）**。把数组看成链表：从下标 `i` 连一条边到 `nums[i]`。因为有重复值，必然有两个下标指向同一个值 → 形成**环**，环的入口就是重复数。
用 141/142 题的找环套路：快慢指针先相遇，再让一个指针回到起点，同速前进，相遇点即环入口（重复数）。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public int findDuplicate(int[] nums) {
        // 把数组当链表：i -> nums[i]。有重复值 ⇒ 有环
        int slow = nums[0], fast = nums[0];
        do {
            slow = nums[slow];          // 慢走一步
            fast = nums[nums[fast]];    // 快走两步
        } while (slow != fast);         // 第一次相遇

        // 找环入口：一个从头、一个从相遇点，同速前进
        slow = nums[0];
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;                    // 环入口 = 重复的数
    }
}
```

### 小结 / 语法点
- 巧妙之处：**把"找重复数"转化为"链表找环入口"**，复用 142 题套路。
- 满足"不改数组 + O(1) 空间"两个苛刻要求（排序会改数组，哈希要 O(n) 空间）。
- `do-while` 保证 slow、fast 先各走一步再比较（初始都在 nums[0]）。

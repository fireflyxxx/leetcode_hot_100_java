## 1. 两数之和（Two Sum）· Easy

🔗 https://leetcode.cn/problems/two-sum/

### 题目大意
给一个整数数组 `nums` 和目标值 `target`，返回**两个数的下标**，使它们的和为 `target`。假设有且仅有一个答案，同一元素不能用两次。

### 思路
- **暴力**：两层循环枚举所有 `(i, j)`，O(n²)。
- **哈希优化**：遍历到 `nums[i]` 时，它需要的"另一半"是 `target - nums[i]`。如果这个另一半**之前出现过**，答案就找到了。所以用一个哈希表记录"**值 → 下标**"，边遍历边查。
- 为什么是边遍历边查、而不是先建完整张表？因为这样能天然保证两个下标不同，且只需一趟。

把它想成：每来一个数，先问"在我前面有没有人正好能和我凑成 target？"有就配对，没有就把自己登记进表，等后面的人来找。

### 复杂度
- 时间 O(n)：每个元素入表、查表都是 O(1)。
- 空间 O(n)：哈希表最多存 n 个元素。

### 代码
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 哈希表：key = 数值，value = 该数值的下标
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int need = target - nums[i];   // 当前数需要配对的“另一半”

            // 如果之前已经登记过这个“另一半”，直接返回两个下标
            if (map.containsKey(need)) {
                return new int[]{map.get(need), i};
            }

            // 没找到配对，就把当前数和它的下标登记进表，供后面的数来查
            map.put(nums[i], i);
        }

        // 题目保证有解，正常不会走到这里
        return new int[0];
    }
}
```

### 小结 / 语法点
- `map.containsKey` + `map.get` 是配对型哈希的标准搭配。
- 返回数组用 `new int[]{a, b}` 的字面量写法。
- **先查后存**的顺序保证了不会用到同一个元素两次。

## 49. 字母异位词分组（Group Anagrams）· Medium

🔗 https://leetcode.cn/problems/group-anagrams/

### 题目大意
给一组字符串，把**互为字母异位词**（字母相同、顺序不同，如 `"eat"`/`"tea"`/`"ate"`）的分到同一组，返回所有分组。

### 思路
关键是给每个字符串找一个**"指纹"**，让异位词的指纹相同，然后用哈希表按指纹分组（指纹做 key，同组字符串列表做 value）。

两种生成指纹的方法：
1. **排序法**：把字符串的字符排序，异位词排序后必然得到相同字符串。如 `"eat"`→`"aet"`，`"tea"`→`"aet"`。简单直观。
2. **计数法**：统计 26 个字母出现次数，拼成一个字符串当 key（如 `"a1e1t1..."`）。省去排序，理论更快。

下面给排序法（最常用、最好记），计数法作为备选。

### 复杂度（设字符串个数 n，最长长度 k）
- 排序法：时间 O(n·k·log k)，空间 O(n·k)。
- 计数法：时间 O(n·k)，空间 O(n·k)。

### 代码（排序法）
```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        // key = 排序后的字符串（异位词共享同一 key），value = 该组所有原始字符串
        Map<String, List<String>> map = new HashMap<>();

        for (String s : strs) {
            char[] arr = s.toCharArray(); // 转成字符数组才能排序
            Arrays.sort(arr);             // 原地排序，得到“指纹”
            String key = new String(arr); // char[] 转回 String 作为 key

            // 若该指纹第一次出现，computeIfAbsent 自动建一个空列表，再把 s 加进去
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }

        // map.values() 就是所有分组，包进 ArrayList 返回
        return new ArrayList<>(map.values());
    }
}
```

### 代码（计数法，备选）
```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        for (String s : strs) {
            int[] cnt = new int[26];
            for (char c : s.toCharArray()) cnt[c - 'a']++; // 统计字母频次

            // 把频次数组拼成 key，比如 "#1#0#0...#2..."，用分隔符防止歧义
            StringBuilder sb = new StringBuilder();
            for (int i = 0; i < 26; i++) sb.append('#').append(cnt[i]);
            String key = sb.toString();

            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(map.values());
    }
}
```

### 小结 / 语法点
- `computeIfAbsent(key, k -> new ArrayList<>())`：**value 是集合时的标准写法**，自动处理"key 第一次出现要先建空集合"。
- `char[]` ↔ `String`：`s.toCharArray()` 与 `new String(arr)`。
- 计数法拼 key 一定要加分隔符 `#`，否则 `[1,12]` 和 `[11,2]` 会混。

## 128. 最长连续序列（Longest Consecutive Sequence）· Medium

🔗 https://leetcode.cn/problems/longest-consecutive-sequence/

### 题目大意
给未排序数组，找出**数值连续**的最长序列长度（元素在数组中顺序无关）。要求 **O(n)** 时间。
例：`[100, 4, 200, 1, 3, 2]` → 连续序列 `1,2,3,4`，返回 `4`。

### 思路
排序能做但是 O(n log n)，不满足要求。用哈希集合做到 O(n)：

1. 把所有数丢进 `HashSet`，O(1) 查询某个数在不在。
2. 遍历每个数 `x`，**只在 `x` 是某段连续序列的"起点"时**才向上数长度。怎么判断是起点？——如果 `x - 1` 不在集合里，那 `x` 就是起点。
3. 从起点 `x` 开始，不断查 `x+1, x+2, ...` 在不在，数出这段长度。

**为什么是 O(n) 而不是 O(n²)？** 关键在第 2 步的"只从起点数"：每段连续序列只会被它的起点触发一次内层 while，每个元素最多被内层访问一次，总访问次数是 O(n)。

### 复杂度
- 时间 O(n)：每个元素最多被外层和内层各访问一次。
- 空间 O(n)：哈希集合。

### 代码
```java
class Solution {
    public int longestConsecutive(int[] nums) {
        // 1) 全部入集合，去重 + O(1) 查询
        Set<Integer> set = new HashSet<>();
        for (int x : nums) set.add(x);

        int longest = 0;

        // 2) 遍历集合（用 set 遍历，重复值只算一次）
        for (int x : set) {
            // 只在 x 是连续序列的“起点”时才开始数：x-1 不存在 ⇒ x 是起点
            if (!set.contains(x - 1)) {
                int cur = x;     // 当前序列正在数到的值
                int len = 1;     // 当前序列长度

                // 不断向上找 cur+1，能找到就延长
                while (set.contains(cur + 1)) {
                    cur++;
                    len++;
                }
                longest = Math.max(longest, len); // 更新全局最长
            }
        }
        return longest;
    }
}
```

### 小结 / 语法点
- **"只从起点扩展"是把 O(n²) 降成 O(n) 的核心剪枝**，一定要理解 `if (!set.contains(x - 1))` 这一句。
- 遍历用 `for (int x : set)` 而非原数组，避免重复值重复计算。
- 用 `HashSet` 不用 `HashMap`，因为只关心"在不在"，不需要附加 value。

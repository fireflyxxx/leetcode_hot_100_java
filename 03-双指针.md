## 283. 移动零（Move Zeroes）· Easy

🔗 https://leetcode.cn/problems/move-zeroes/

### 题目大意
把数组里所有 `0` 移到末尾，保持非零元素的**相对顺序**，且必须**原地**操作。
例：`[0,1,0,3,12]` → `[1,3,12,0,0]`。

### 思路
**快慢指针（同向）**。`slow` 指向"下一个非零元素应该放的位置"，`fast` 向前扫：
- 遇到非零元素，就和 `slow` 处交换，然后 `slow++`。
- 遇到 0 就跳过（`fast++`，`slow` 不动）。

这样 `slow` 左边永远是已经排好的非零元素，遍历完后 `slow` 及之后自然都是 0。交换能保证非零元素相对顺序不变。

### 复杂度
- 时间 O(n)，一趟遍历。
- 空间 O(1)，原地交换。

### 代码
```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0; // [0, slow) 区间是已经处理好的非零元素

        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                // 把非零元素交换到 slow 位置（slow<=fast，不会破坏未处理区）
                int tmp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = tmp;
                slow++;       // 非零区扩大一格
            }
            // nums[fast]==0 时什么都不做，fast 继续前进
        }
    }
}
```

### 小结 / 语法点
- 快慢指针处理"原地保留/移除"类问题的通用骨架：`slow` 是写指针，`fast` 是读指针。
- 这里用交换而不是覆盖，能保证 0 被换到后面，无需最后补 0。

## 11. 盛最多水的容器（Container With Most Water）· Medium

🔗 https://leetcode.cn/problems/container-with-most-water/

### 题目大意
数组 `height[i]` 表示第 i 根竖线的高度。任选两根线和 x 轴围成容器，求能装的**最大水量**。
水量 = `两线中较矮的高度 × 两线间距`。

### 思路
**对撞指针**。左右指针放两端，此时**底最宽**。每次：
- 计算当前面积 `min(左高, 右高) × 宽度`，更新最大值。
- **移动较矮的那根线**向内。

为什么移动矮的？因为面积受限于较矮的线。若移动高的，宽度减小、瓶颈（矮线）不变，面积只会变小；只有移动矮的才**有可能**遇到更高的线、让面积变大。所以贪心地舍弃矮线。

### 复杂度
- 时间 O(n)，左右指针总共移动 n 步。
- 空间 O(1)。

### 代码
```java
class Solution {
    public int maxArea(int[] height) {
        int l = 0, r = height.length - 1; // 左右指针从两端开始（底最宽）
        int max = 0;

        while (l < r) {
            // 面积 = 较矮的高度 × 宽度
            int area = Math.min(height[l], height[r]) * (r - l);
            max = Math.max(max, area);

            // 移动较矮的一侧（保留较高的，才有机会变大）
            if (height[l] < height[r]) {
                l++;
            } else {
                r--;
            }
        }
        return max;
    }
}
```

### 小结 / 语法点
- 对撞指针的精髓是"**每一步排除一个不可能更优的选择**"，把 O(n²) 枚举降为 O(n)。
- 关键在想通"为什么移动矮的"——这是贪心的正确性证明。

## 15. 三数之和（3Sum）· Medium

🔗 https://leetcode.cn/problems/3sum/

### 题目大意
找出数组中所有**和为 0** 且**不重复**的三元组 `[a, b, c]`。
例：`[-1,0,1,2,-1,-4]` → `[[-1,-1,2],[-1,0,1]]`。

### 思路
**排序 + 固定一个数 + 对撞指针**：
1. 先排序，让数组有序（这样才能用双指针，也方便去重）。
2. 外层固定第一个数 `nums[i]`，问题转化为在 `i` 右边找两个数和为 `-nums[i]`（即"两数之和"，有序数组用对撞指针 O(n)）。
3. 内层左右指针 `l, r`：和太小就 `l++`（变大），太大就 `r--`（变小），正好就记录。

**去重是难点**，三处都要处理：
- 外层：`nums[i] == nums[i-1]` 时跳过（同一个 a 不重复枚举）。
- 找到一组解后：跳过和当前 `nums[l]` 相同的、和当前 `nums[r]` 相同的。

剪枝：排序后若 `nums[i] > 0`，三个最小数都 > 0，不可能和为 0，直接 break。

### 复杂度
- 时间 O(n²)：外层 O(n)，内层双指针 O(n)。排序 O(n log n) 被吃掉。
- 空间 O(log n)（排序），不计返回结果。

### 代码
```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);                       // 关键第一步：排序
        List<List<Integer>> res = new ArrayList<>();
        int n = nums.length;

        for (int i = 0; i < n - 2; i++) {
            // 剪枝：最小的数都 > 0，三数之和不可能为 0
            if (nums[i] > 0) break;
            // 去重 a：跳过重复的第一个数
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            int l = i + 1, r = n - 1;            // 在 i 右边用对撞指针
            while (l < r) {
                int sum = nums[i] + nums[l] + nums[r];
                if (sum == 0) {
                    // 记录这一组解
                    res.add(Arrays.asList(nums[i], nums[l], nums[r]));
                    // 去重 b、c：跳过与当前相同的相邻元素
                    while (l < r && nums[l] == nums[l + 1]) l++;
                    while (l < r && nums[r] == nums[r - 1]) r--;
                    // 收缩到下一对不同的数
                    l++;
                    r--;
                } else if (sum < 0) {
                    l++;   // 和偏小，左指针右移让和变大
                } else {
                    r--;   // 和偏大，右指针左移让和变小
                }
            }
        }
        return res;
    }
}
```

### 小结 / 语法点
- **"排序 + 双指针"是 N 数之和的通用框架**（四数之和再套一层循环）。
- 去重的三处位置要记牢：外层 a 一处，找到解后 b、c 各一处。
- `Arrays.asList(...)` 快速构造定长 List 作为一组解。

## 42. 接雨水（Trapping Rain Water）· Hard

🔗 https://leetcode.cn/problems/trapping-rain-water/

### 题目大意
`height[i]` 是柱子高度，柱子间能接多少雨水。
例：`[0,1,0,2,1,0,1,3,2,1,2,1]` → `6`。

### 思路
**位置 i 能接的水 = min(左边最高, 右边最高) - height[i]**（木桶效应，两边短板决定水位）。

- **朴素**：对每个 i 都向左右扫一遍求最高，O(n²)。
- **DP 预处理**：先求出每个位置的 `leftMax[]` 和 `rightMax[]`，再一趟累加，O(n) 时间 O(n) 空间。
- **双指针（最优）**：用 `leftMax`、`rightMax` 两个变量边走边维护，把空间降到 O(1)。

**双指针为什么对？** 左右指针从两端往中间走，维护"目前见过的左侧最高 `leftMax`、右侧最高 `rightMax`"。当 `leftMax < rightMax` 时，**位置 l 的水位一定由 leftMax 决定**——因为右边存在一个比 leftMax 还高的柱子（rightMax）兜着，l 处水位不会超过 leftMax。于是直接 `water += leftMax - height[l]`，再 `l++`。反之处理右边。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码（双指针，最优）
```java
class Solution {
    public int trap(int[] height) {
        int l = 0, r = height.length - 1;
        int leftMax = 0, rightMax = 0; // 分别记录 [0..l]、[r..n-1] 的最高柱
        int water = 0;

        while (l < r) {
            // 先把当前柱纳入对应的最高值
            leftMax = Math.max(leftMax, height[l]);
            rightMax = Math.max(rightMax, height[r]);

            // 谁的最高值小，谁那一侧的水位就能确定（被另一侧更高的柱兜住）
            if (leftMax < rightMax) {
                water += leftMax - height[l]; // l 处水位由 leftMax 决定
                l++;
            } else {
                water += rightMax - height[r];// r 处水位由 rightMax 决定
                r--;
            }
        }
        return water;
    }
}
```

### 代码（DP 预处理，更直观，建议先理解这版）
```java
class Solution {
    public int trap(int[] height) {
        int n = height.length;
        if (n == 0) return 0;
        int[] leftMax = new int[n];   // leftMax[i] = [0..i] 的最高柱
        int[] rightMax = new int[n];  // rightMax[i] = [i..n-1] 的最高柱

        leftMax[0] = height[0];
        for (int i = 1; i < n; i++)
            leftMax[i] = Math.max(leftMax[i - 1], height[i]);

        rightMax[n - 1] = height[n - 1];
        for (int i = n - 2; i >= 0; i--)
            rightMax[i] = Math.max(rightMax[i + 1], height[i]);

        int water = 0;
        for (int i = 0; i < n; i++)
            water += Math.min(leftMax[i], rightMax[i]) - height[i]; // 木桶效应
        return water;
    }
}
```

### 小结 / 语法点
- 核心公式 `min(leftMax, rightMax) - height[i]` 一定要记住，三种解法都基于它。
- 双指针把 DP 的两个数组优化成两个变量，是"空间优化"的经典案例。
- 接雨水还有**单调栈**解法（见套路总纲），三种都掌握最好。

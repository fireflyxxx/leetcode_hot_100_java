## 62. 不同路径（Unique Paths）· Medium

🔗 https://leetcode.cn/problems/unique-paths/

### 题目大意
`m×n` 网格，机器人从左上角走到右下角，每次只能向右或向下，问有多少条不同路径。

### 思路
**网格 DP**。`dp[i][j]` = 到达 `(i,j)` 的路径数。只能从上方或左方来，所以 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`。第一行、第一列只有一条路（一直向右/向下），初始化为 1。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)（可优化到 O(n)）。

### 代码
```java
class Solution {
    public int uniquePaths(int m, int n) {
        int[][] dp = new int[m][n];
        for (int i = 0; i < m; i++) dp[i][0] = 1;  // 第一列：只能一直向下
        for (int j = 0; j < n; j++) dp[0][j] = 1;  // 第一行：只能一直向右
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[i][j] = dp[i - 1][j] + dp[i][j - 1]; // 上方 + 左方
            }
        }
        return dp[m - 1][n - 1];
    }
}
```

### 小结 / 语法点
- 网格路径 DP 模板：`dp[i][j]` 来自上和左。
- 第一行第一列是 base case，要先初始化。

## 64. 最小路径和（Minimum Path Sum）· Medium

🔗 https://leetcode.cn/problems/minimum-path-sum/

### 题目大意
`m×n` 网格每格有非负数，从左上走到右下（只能右/下），求路径上数字总和的最小值。

### 思路
和 62 同框架，转移取 min 并加上当前格的值：`dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`。第一行、第一列只能由唯一方向累加。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)。

### 代码
```java
class Solution {
    public int minPathSum(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        int[][] dp = new int[m][n];
        dp[0][0] = grid[0][0];
        for (int i = 1; i < m; i++) dp[i][0] = dp[i - 1][0] + grid[i][0]; // 第一列累加
        for (int j = 1; j < n; j++) dp[0][j] = dp[0][j - 1] + grid[0][j]; // 第一行累加
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[i][j] = Math.min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j];
            }
        }
        return dp[m - 1][n - 1];
    }
}
```

### 小结 / 语法点
- 与 62 唯一区别：`+ -> min` 并叠加 `grid[i][j]`。
- 第一行第一列要单独累加初始化（不能只填 grid 值）。

## 5. 最长回文子串（Longest Palindromic Substring）· Medium

🔗 https://leetcode.cn/problems/longest-palindromic-substring/

### 题目大意
返回字符串中**最长的回文子串**。
例：`"babad"` → `"bab"`（或 `"aba"`）。

### 思路
**中心扩展**（比二维 DP 更省空间，最常用）。回文中心要么是一个字符（奇数长度），要么是两个字符之间（偶数长度）。枚举每个中心，向两边扩展直到不再对称，记录最长。
（也可用 `dp[i][j]` 表示 `s[i..j]` 是否回文，转移 `dp[i][j] = (s[i]==s[j]) && dp[i+1][j-1]`，但 O(n²) 空间，这里用中心扩展。）

### 复杂度
- 时间 O(n²)，空间 O(1)。

### 代码
```java
class Solution {
    public String longestPalindrome(String s) {
        if (s == null || s.length() < 1) return "";
        int start = 0, end = 0;                    // 记录最长回文的左右边界
        for (int i = 0; i < s.length(); i++) {
            int len1 = expand(s, i, i);            // 以 i 为中心（奇数长度）
            int len2 = expand(s, i, i + 1);        // 以 i、i+1 之间为中心（偶数长度）
            int len = Math.max(len1, len2);
            if (len > end - start + 1) {           // 更新最长
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }
        return s.substring(start, end + 1);
    }
    // 从 (l, r) 向两边扩展，返回回文长度
    private int expand(String s, int l, int r) {
        while (l >= 0 && r < s.length() && s.charAt(l) == s.charAt(r)) {
            l--;
            r++;
        }
        return r - l - 1;                          // 退出时多走了一步，长度 = r-l-1
    }
}
```

### 小结 / 语法点
- **中心扩展**要考虑奇、偶两种中心，各扩一次取最大。
- `expand` 返回 `r - l - 1`（循环结束时 l、r 已各超出一格）。
- 由 len 反推边界：`start = i - (len-1)/2`，`end = i + len/2`。

## 1143. 最长公共子序列（Longest Common Subsequence）· Medium

🔗 https://leetcode.cn/problems/longest-common-subsequence/

### 题目大意
求两个字符串的最长**公共子序列**长度（子序列不要求连续，但相对顺序一致）。

### 思路
**二维字符串 DP**。`dp[i][j]` = `text1` 前 i 个字符与 `text2` 前 j 个字符的 LCS 长度。
- 若 `text1[i-1] == text2[j-1]`：这对字符可加入公共子序列，`dp[i][j] = dp[i-1][j-1] + 1`。
- 否则：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`（放弃其中一个字符）。
下标用 1-based，`dp[0][*]=dp[*][0]=0`。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)。

### 代码
```java
class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();
        int[][] dp = new int[m + 1][n + 1];        // dp[0][*]、dp[*][0] 默认 0
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;     // 字符相同，对角线 +1
                } else {
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]); // 取上 / 左较大
                }
            }
        }
        return dp[m][n];
    }
}
```

### 小结 / 语法点
- 两个序列对齐 → 二维 DP，**字符相等看对角线 +1，不等取上/左最大**。
- 用 1-based 下标，`charAt(i-1)` 对应 `dp[i]`，避免边界判断。

## 72. 编辑距离（Edit Distance）· Medium

🔗 https://leetcode.cn/problems/edit-distance/

### 题目大意
求把 `word1` 变成 `word2` 的最少操作数（可**插入、删除、替换**一个字符）。

### 思路
**二维 DP**。`dp[i][j]` = 把 `word1` 前 i 个变成 `word2` 前 j 个的最少操作数。
- 若末字符相同：`dp[i][j] = dp[i-1][j-1]`（不用操作）。
- 否则取三种操作的最小 + 1：
  - `dp[i-1][j] + 1`：删除 word1 的第 i 个字符；
  - `dp[i][j-1] + 1`：插入一个字符匹配 word2 的第 j 个；
  - `dp[i-1][j-1] + 1`：替换 word1 第 i 个为 word2 第 j 个。
- base：`dp[i][0] = i`（全删），`dp[0][j] = j`（全插）。

### 复杂度
- 时间 O(m·n)，空间 O(m·n)。

### 代码
```java
class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 0; i <= m; i++) dp[i][0] = i;   // word1 前 i 个删空，需 i 步
        for (int j = 0; j <= n; j++) dp[0][j] = j;   // 空插成 word2 前 j 个，需 j 步
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    dp[i][j] = dp[i - 1][j - 1];      // 末字符相同，无需操作
                } else {
                    // 删、插、替 三选一，+1
                    dp[i][j] = Math.min(Math.min(dp[i - 1][j], dp[i][j - 1]), dp[i - 1][j - 1]) + 1;
                }
            }
        }
        return dp[m][n];
    }
}
```

### 小结 / 语法点
- 编辑距离是二维 DP 的"天花板题"，三个来源分别对应**删、插、替**。
- base case：一个串为空时，操作数 = 另一个串的长度。
- 与 LCS 结构几乎一致，区别在转移项和 base。

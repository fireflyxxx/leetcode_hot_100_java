# 09 · 二叉树（15 题 · 重点板块）

> 二叉树是 Hot100 题量最大的板块。核心就两种遍历框架：**递归 DFS**（前/中/后序）和 **层序 BFS**。
> 大多数题的套路是："想清楚每个节点需要从左右子树拿什么信息，再决定用前序（自顶向下传参）还是后序（自底向上汇总）。"

> **节点定义**（LeetCode 已给）：
> ```java
> public class TreeNode {
>     int val;
>     TreeNode left, right;
>     TreeNode() {}
>     TreeNode(int val) { this.val = val; }
>     TreeNode(int val, TreeNode left, TreeNode right) {
>         this.val = val; this.left = left; this.right = right;
>     }
> }
> ```

> **两个通用框架**
> ```java
> // DFS 递归框架
> void dfs(TreeNode node) {
>     if (node == null) return;     // 递归出口
>     // 前序位置：处理 node（自顶向下）
>     dfs(node.left);
>     // 中序位置：BST 中序 = 升序
>     dfs(node.right);
>     // 后序位置：左右子树都处理完（自底向上汇总）
> }
> // BFS 层序框架
> Queue<TreeNode> q = new LinkedList<>(); q.offer(root);
> while (!q.isEmpty()) {
>     int size = q.size();          // 锁定当前层节点数
>     for (int i = 0; i < size; i++) {
>         TreeNode node = q.poll();
>         if (node.left != null) q.offer(node.left);
>         if (node.right != null) q.offer(node.right);
>     }
> }
> ```

## 94. 二叉树的中序遍历（Inorder Traversal）· Easy

🔗 https://leetcode.cn/problems/binary-tree-inorder-traversal/

### 题目大意
返回中序遍历（左 → 根 → 右）的节点值列表。

### 思路
递归最直观：先递归左子树，再访问根，再递归右子树。
也要会**迭代版**（用栈模拟）：一路向左把节点压栈，弹出即访问，再转向其右子树。面试常考迭代写法。

### 复杂度
- 时间 O(n)，空间 O(n)（递归栈/显式栈）。

### 代码（递归）
```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> res = new ArrayList<>();
        inorder(root, res);
        return res;
    }
    private void inorder(TreeNode node, List<Integer> res) {
        if (node == null) return;
        inorder(node.left, res);   // 左
        res.add(node.val);         // 根
        inorder(node.right, res);  // 右
    }
}
```

### 代码（迭代，用栈）
```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> res = new ArrayList<>();
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode cur = root;
        while (cur != null || !stack.isEmpty()) {
            while (cur != null) {      // 一路向左，沿途压栈
                stack.push(cur);
                cur = cur.left;
            }
            cur = stack.pop();         // 弹出最左节点
            res.add(cur.val);          // 访问
            cur = cur.right;           // 转向右子树
        }
        return res;
    }
}
```

### 小结 / 语法点
- 中序 = 左根右；前序把 `res.add` 提到递归左子树前，后序放到递归右子树后。
- BST 的中序遍历是**升序**序列——这是 98、230 题的关键。

## 104. 二叉树的最大深度（Maximum Depth）· Easy

🔗 https://leetcode.cn/problems/maximum-depth-of-binary-tree/

### 题目大意
返回根到最远叶子的节点数（深度）。

### 思路
**后序递归**：一棵树的深度 = max(左子树深度, 右子树深度) + 1。空树深度 0。
这是"自底向上汇总信息"的最简单例子。

### 复杂度
- 时间 O(n)，空间 O(h)（h 为树高）。

### 代码
```java
class Solution {
    public int maxDepth(TreeNode root) {
        if (root == null) return 0;                 // 空树深度 0
        int left = maxDepth(root.left);
        int right = maxDepth(root.right);
        return Math.max(left, right) + 1;           // 加上当前层
    }
}
```

### 小结 / 语法点
- 树形递归的"骨架题"：先递归拿左右子树结果，再合并。
- 类似可解 111 最小深度、110 平衡二叉树。

## 226. 翻转二叉树（Invert Binary Tree）· Easy

🔗 https://leetcode.cn/problems/invert-binary-tree/

### 题目大意
把二叉树左右翻转（镜像）。

### 思路
递归翻转左右子树，然后交换当前节点的左右孩子。前序或后序都行，关键是"每个节点都交换一次左右孩子"。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    public TreeNode invertTree(TreeNode root) {
        if (root == null) return null;
        TreeNode left = invertTree(root.left);   // 先翻转左子树
        TreeNode right = invertTree(root.right); // 再翻转右子树
        root.left = right;                       // 交换
        root.right = left;
        return root;
    }
}
```

### 小结 / 语法点
- 一行核心：交换左右孩子，递归到底。
- 用临时变量接住递归结果再交换，避免覆盖。

## 101. 对称二叉树（Symmetric Tree）· Easy

🔗 https://leetcode.cn/problems/symmetric-tree/

### 题目大意
判断一棵树是否**轴对称**（镜像）。

### 思路
不是判断单棵树，而是判断"**左子树和右子树是否互为镜像**"。定义辅助函数 `check(a, b)`：
- 都空 → 对称；只有一个空 → 不对称；
- 值不等 → 不对称；
- 递归比较 `a.left` 与 `b.right`、`a.right` 与 `b.left`（注意是交叉比较）。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    public boolean isSymmetric(TreeNode root) {
        if (root == null) return true;
        return check(root.left, root.right);
    }
    private boolean check(TreeNode a, TreeNode b) {
        if (a == null && b == null) return true;  // 都空，对称
        if (a == null || b == null) return false; // 一空一非空，不对称
        // 值相等，且“外侧 vs 外侧、内侧 vs 内侧”交叉镜像
        return a.val == b.val
            && check(a.left, b.right)
            && check(a.right, b.left);
    }
}
```

### 小结 / 语法点
- 关键是**交叉比较** `a.left↔b.right`、`a.right↔b.left`。
- "判断两棵树关系"的题（100 相同的树）都用双参数递归。

## 543. 二叉树的直径（Diameter of Binary Tree）· Easy

🔗 https://leetcode.cn/problems/diameter-of-binary-tree/

### 题目大意
返回任意两节点间最长路径的**边数**（不一定经过根）。

### 思路
**后序递归 + 全局变量**。对每个节点，"经过它的最长路径长度" = 左子树深度 + 右子树深度。遍历所有节点取最大值。
递归函数返回的是"以该节点为顶的最大深度"，同时在递归中**顺便更新全局直径**。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    private int diameter = 0; // 全局记录最长路径（边数）

    public int diameterOfBinaryTree(TreeNode root) {
        depth(root);
        return diameter;
    }

    // 返回以 node 为根的最大深度（节点数）
    private int depth(TreeNode node) {
        if (node == null) return 0;
        int l = depth(node.left);
        int r = depth(node.right);
        diameter = Math.max(diameter, l + r); // 经过 node 的路径边数 = 左深 + 右深
        return Math.max(l, r) + 1;            // 向上返回更深的一支 + 1
    }
}
```

### 小结 / 语法点
- 套路：**递归返回一个值（深度），但用全局变量记录另一个量（直径）**。124 最大路径和同款思路。
- 直径按"边数"算，正好等于左右子树深度之和。

## 102. 二叉树的层序遍历（Level Order Traversal）· Medium

🔗 https://leetcode.cn/problems/binary-tree-level-order-traversal/

### 题目大意
按层从上到下、每层从左到右返回节点值，每层一个子列表。

### 思路
**BFS + 按层切分**。用队列，每轮循环开始先记录当前队列大小 `size`（即本层节点数），只处理这 `size` 个，把它们的孩子入队，收集成一个 level 列表。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public List<List<Integer>> levelOrder(TreeNode root) {
        List<List<Integer>> res = new ArrayList<>();
        if (root == null) return res;
        Queue<TreeNode> q = new LinkedList<>();
        q.offer(root);

        while (!q.isEmpty()) {
            int size = q.size();              // 本层节点数（关键！先存下来）
            List<Integer> level = new ArrayList<>();
            for (int i = 0; i < size; i++) {
                TreeNode node = q.poll();
                level.add(node.val);
                if (node.left != null) q.offer(node.left);
                if (node.right != null) q.offer(node.right);
            }
            res.add(level);                   // 一层收集完
        }
        return res;
    }
}
```

### 小结 / 语法点
- **`int size = q.size()` 锁定本层节点数**是 BFS 分层的精髓，199 右视图、103 锯齿遍历都靠它。
- `Queue<TreeNode> q = new LinkedList<>()`，用 `offer/poll`。

## 108. 将有序数组转换为二叉搜索树（Convert Sorted Array to BST）· Easy

🔗 https://leetcode.cn/problems/convert-sorted-array-to-binary-search-tree/

### 题目大意
把升序数组转成一棵**高度平衡**的 BST。

### 思路
**分治**。取数组中点作为根（保证左右数量均衡 → 平衡），左半边递归建左子树，右半边递归建右子树。因为数组有序，中点天然满足 BST 性质。

### 复杂度
- 时间 O(n)，空间 O(log n)（递归栈）。

### 代码
```java
class Solution {
    public TreeNode sortedArrayToBST(int[] nums) {
        return build(nums, 0, nums.length - 1);
    }
    // 用 nums[lo..hi] 构造平衡 BST
    private TreeNode build(int[] nums, int lo, int hi) {
        if (lo > hi) return null;
        int mid = lo + (hi - lo) / 2;        // 取中点当根，保证平衡
        TreeNode root = new TreeNode(nums[mid]);
        root.left = build(nums, lo, mid - 1);   // 左半建左子树
        root.right = build(nums, mid + 1, hi);  // 右半建右子树
        return root;
    }
}
```

### 小结 / 语法点
- "**有序数组 + 平衡 BST**" → 取中点分治。
- 中点用 `lo + (hi - lo) / 2` 防溢出。

## 98. 验证二叉搜索树（Validate BST）· Medium

🔗 https://leetcode.cn/problems/validate-binary-search-tree/

### 题目大意
判断一棵树是否为合法 BST（左子树所有值 < 根 < 右子树所有值，**整棵子树**都要满足）。

### 思路
常见错误是只比较节点和它的直接孩子——必须保证**整棵左子树都小于根**。
**方法一（上下界）**：递归时传入当前节点允许的取值区间 `(min, max)`，每下一层收紧区间。
**方法二（中序遍历）**：BST 的中序遍历严格递增，遍历时检查是否一直比前一个大。
下面用上下界法（注意用 `long` 防止节点值取到 `Integer.MIN/MAX_VALUE` 的边界问题）。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        return valid(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }
    // node 的值必须严格落在开区间 (min, max) 内
    private boolean valid(TreeNode node, long min, long max) {
        if (node == null) return true;
        if (node.val <= min || node.val >= max) return false;
        // 左子树上界收紧为 node.val，右子树下界收紧为 node.val
        return valid(node.left, min, node.val)
            && valid(node.right, node.val, max);
    }
}
```

### 小结 / 语法点
- BST 验证要传**上下界**，不能只比父子；区间逐层收紧。
- 用 `long` 边界避免节点值等于 `Integer.MIN_VALUE/MAX_VALUE` 时误判。

## 230. 二叉搜索树中第 K 小的元素（Kth Smallest in BST）· Medium

🔗 https://leetcode.cn/problems/kth-smallest-element-in-a-bst/

### 题目大意
返回 BST 中第 `k` 小的元素（1-indexed）。

### 思路
**BST 中序遍历 = 升序**，所以中序走到第 k 个节点就是答案。用迭代中序遍历，每弹出一个节点就 `k--`，减到 0 时返回。比递归收集整个列表更省（可提前停止）。

### 复杂度
- 时间 O(h + k)，空间 O(h)。

### 代码
```java
class Solution {
    public int kthSmallest(TreeNode root, int k) {
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode cur = root;
        while (cur != null || !stack.isEmpty()) {
            while (cur != null) {        // 一路向左压栈
                stack.push(cur);
                cur = cur.left;
            }
            cur = stack.pop();           // 中序访问到的下一个（升序）
            if (--k == 0) return cur.val; // 第 k 小
            cur = cur.right;
        }
        return -1;                       // 题目保证有解
    }
}
```

### 小结 / 语法点
- 见到"**BST + 第 K 小/大**"立刻想到**中序遍历**。
- 迭代中序能在数到第 k 个时立即返回，不必遍历整棵树。

## 199. 二叉树的右视图（Right Side View）· Medium

🔗 https://leetcode.cn/problems/binary-tree-right-side-view/

### 题目大意
站在树的右侧，从上到下能看到的节点值（即每层最右边的节点）。

### 思路
**BFS 分层**，取每一层的**最后一个**节点（`i == size - 1`）。复用 102 的分层框架。
（也可用 DFS"根→右→左"的顺序，每层第一个被访问的就是右视图，记录深度去重。）

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    public List<Integer> rightSideView(TreeNode root) {
        List<Integer> res = new ArrayList<>();
        if (root == null) return res;
        Queue<TreeNode> q = new LinkedList<>();
        q.offer(root);
        while (!q.isEmpty()) {
            int size = q.size();
            for (int i = 0; i < size; i++) {
                TreeNode node = q.poll();
                if (i == size - 1) res.add(node.val); // 本层最后一个 = 右视图
                if (node.left != null) q.offer(node.left);
                if (node.right != null) q.offer(node.right);
            }
        }
        return res;
    }
}
```

### 小结 / 语法点
- 在 102 分层框架上，只多一句 `if (i == size - 1)` 取每层最后一个。
- 左视图就是取 `i == 0`。

## 114. 二叉树展开为链表（Flatten Binary Tree to Linked List）· Medium

🔗 https://leetcode.cn/problems/flatten-binary-tree-to-linked-list/

### 题目大意
把二叉树原地展开成一条**只用 right 指针**的链表，顺序为**前序遍历**，left 全置 null。

### 思路
**前驱拼接法（O(1) 空间，最优雅）**：对每个有左子树的节点 `cur`：
1. 找到左子树的**最右节点** `prev`（它是前序中左子树最后一个节点）。
2. 把 `cur` 的右子树接到 `prev.right` 上。
3. 把左子树整体挪到右边（`cur.right = cur.left`），`cur.left = null`。
4. `cur` 前进到 `cur.right` 继续。
这样不断把左子树"插入"到右链中，最终形成前序链表。

### 复杂度
- 时间 O(n)，空间 O(1)。

### 代码
```java
class Solution {
    public void flatten(TreeNode root) {
        TreeNode cur = root;
        while (cur != null) {
            if (cur.left != null) {
                // 1) 找左子树的最右节点（前序里左子树的末尾）
                TreeNode prev = cur.left;
                while (prev.right != null) prev = prev.right;
                // 2) 原来的右子树接到该前驱后面
                prev.right = cur.right;
                // 3) 左子树移到右边，左指针清空
                cur.right = cur.left;
                cur.left = null;
            }
            cur = cur.right; // 4) 继续处理下一个
        }
    }
}
```

### 小结 / 语法点
- 核心：**把当前右子树挂到左子树的最右端**，再把左子树搬到右边。
- 这是 Morris 思想的应用，O(1) 空间，无需递归或栈。

## 105. 从前序与中序遍历序列构造二叉树（Build Tree from Preorder & Inorder）· Medium

🔗 https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/

### 题目大意
给前序和中序遍历序列（无重复值），重建二叉树。

### 思路
**分治 + 哈希定位**。
- 前序的第一个元素一定是**根**。
- 在中序中找到根的位置，它**左边是左子树的中序、右边是右子树的中序**，由此知道左子树有多少个节点 `leftSize`。
- 前序中紧跟根之后的 `leftSize` 个是左子树前序，再后面是右子树前序。
- 递归构造。用哈希表存"中序值 → 下标"，O(1) 定位根，免去每次线性查找。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    private Map<Integer, Integer> inIdx = new HashMap<>(); // 中序值 -> 下标

    public TreeNode buildTree(int[] preorder, int[] inorder) {
        for (int i = 0; i < inorder.length; i++) inIdx.put(inorder[i], i);
        return build(preorder, 0, preorder.length - 1, 0);
    }

    // preL/preR：当前子树在前序中的区间；inL：当前子树在中序中的起点
    private TreeNode build(int[] pre, int preL, int preR, int inL) {
        if (preL > preR) return null;
        int rootVal = pre[preL];               // 前序首元素是根
        TreeNode root = new TreeNode(rootVal);

        int inRoot = inIdx.get(rootVal);        // 根在中序中的位置
        int leftSize = inRoot - inL;            // 左子树节点个数

        // 左子树：前序 [preL+1, preL+leftSize]，中序起点 inL
        root.left = build(pre, preL + 1, preL + leftSize, inL);
        // 右子树：前序 [preL+leftSize+1, preR]，中序起点 inRoot+1
        root.right = build(pre, preL + leftSize + 1, preR, inRoot + 1);
        return root;
    }
}
```

### 小结 / 语法点
- 口诀：**前序定根，中序分左右**。算出 `leftSize` 是关键。
- 哈希表存中序下标把定位从 O(n) 降到 O(1)，整体 O(n)。

## 437. 路径总和 III（Path Sum III）· Medium

🔗 https://leetcode.cn/problems/path-sum-iii/

### 题目大意
统计路径和等于 `targetSum` 的路径数目。路径方向**向下**（父→子）但**不必从根开始、不必到叶子结束**。

### 思路
**前缀和 + 哈希 + 回溯**（把 560"和为K的子数组"搬到树上）。
- 沿根到当前节点的路径维护"前缀和"。当前前缀和 `cur`，若历史上存在前缀和等于 `cur - target`，则那一段子路径的和正好是 target。
- 用哈希表记录"从根到当前路径上，各前缀和出现的次数"。
- **关键是回溯**：处理完一个节点的子树后，要把它的前缀和计数减回去（因为离开了这条路径）。

### 复杂度
- 时间 O(n)，空间 O(n)。

### 代码
```java
class Solution {
    private int count = 0;
    private Map<Long, Integer> prefix = new HashMap<>(); // 前缀和 -> 次数

    public int pathSum(TreeNode root, int targetSum) {
        prefix.put(0L, 1);                 // 空前缀，和为 0 出现 1 次
        dfs(root, 0L, targetSum);
        return count;
    }

    private void dfs(TreeNode node, long cur, int target) {
        if (node == null) return;
        cur += node.val;                                   // 当前根到 node 的前缀和
        count += prefix.getOrDefault(cur - target, 0);     // 有几个起点能凑出 target
        prefix.merge(cur, 1, Integer::sum);                // 登记当前前缀和

        dfs(node.left, cur, target);
        dfs(node.right, cur, target);

        prefix.merge(cur, -1, Integer::sum);               // 回溯：离开该节点，撤销登记
    }
}
```

### 小结 / 语法点
- "**树上路径和计数**" = 前缀和哈希 + DFS 回溯，是 560 题的树版。
- 用 `long` 防前缀和溢出；**回溯那一行 `merge(cur, -1, ...)` 千万不能漏**。

## 236. 二叉树的最近公共祖先（Lowest Common Ancestor）· Medium

🔗 https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/

### 题目大意
找两个给定节点 `p`、`q` 的最近公共祖先（LCA）。

### 思路
**后序递归**。递归含义：在以 root 为根的子树里找 p、q。
- 若 `root` 为空、或等于 p、或等于 q，直接返回 `root`（找到了其一，或到底了）。
- 递归左右子树，得到 `left`、`right`：
  - 左右**都非空** → p、q 分别在两侧，当前 `root` 就是 LCA。
  - 只有一侧非空 → 返回那一侧（p、q 都在那侧，或只找到一个）。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        // 出口：空，或撞上 p / q，直接上交
        if (root == null || root == p || root == q) return root;

        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);

        if (left != null && right != null) return root; // 左右各找到一个 ⇒ root 是 LCA
        return left != null ? left : right;             // 否则把非空的一侧上交
    }
}
```

### 小结 / 语法点
- 经典后序递归：**"左右都有就是我，否则上交非空那个"**。
- 用 `==` 比较节点引用（题目保证节点唯一）。

## 124. 二叉树中的最大路径和（Binary Tree Maximum Path Sum）· Hard

🔗 https://leetcode.cn/problems/binary-tree-maximum-path-sum/

### 题目大意
路径是任意节点序列（相邻节点有边相连），路径**至少一个节点**、不必经过根。返回最大路径和。

### 思路
**后序递归 + 全局最大值**（和 543 直径同款套路，把"长度"换成"和"）。
定义 `gain(node)` = "从 node 出发**向下走一条分支**能获得的最大和"。
- 左右分支贡献 `left = max(gain(left), 0)`、`right = max(gain(right), 0)`——**负贡献就舍弃（取 0）**。
- 以 node 为"拐点"的路径和 = `node.val + left + right`，用它更新全局最大。
- 但 `gain(node)` 返回给上层时只能选一条分支：`node.val + max(left, right)`（路径不能在 node 分叉后还往上）。

### 复杂度
- 时间 O(n)，空间 O(h)。

### 代码
```java
class Solution {
    private int maxSum = Integer.MIN_VALUE; // 全局最大路径和

    public int maxPathSum(TreeNode root) {
        gain(root);
        return maxSum;
    }

    // 返回以 node 为起点、向下延伸一条分支的最大和
    private int gain(TreeNode node) {
        if (node == null) return 0;
        int left = Math.max(gain(node.left), 0);   // 负的就不要（取 0）
        int right = Math.max(gain(node.right), 0);

        // 以 node 为拐点（左+node+右）的路径，用来更新全局答案
        maxSum = Math.max(maxSum, node.val + left + right);

        // 向上返回只能保留一条分支
        return node.val + Math.max(left, right);
    }
}
```

### 小结 / 语法点
- **"经过拐点更新答案、向上只返回单边"** 是树形 DP 的经典手法（543、124 一脉相承）。
- 负贡献 `Math.max(gain, 0)` 直接砍掉，是本题精髓。
- `maxSum` 初值用 `Integer.MIN_VALUE`，因为节点值可能全为负。

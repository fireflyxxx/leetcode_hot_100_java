## 0. 通用解题流程

1. **读题**：明确输入输出、数据范围、边界（空输入、单元素、负数、重复值）。
2. **看数据量猜复杂度**（见下表）：n ≤ 20 多半是回溯/状压；n ≈ 10⁵ 多半要 O(n) 或 O(n log n)。
3. **想暴力解**：先确保有一个能 AC 的笨办法，再优化。
4. **找瓶颈 → 套路匹配**：哪一步重复计算多？能否用哈希/双指针/前缀和/单调结构把它降下来。
5. **写代码**：先写主干，再补边界。
6. **测边界**：空、单元素、全相同、最大最小、溢出。

## 1. 复杂度速记

| 数据范围 n | 可接受复杂度 | 常见解法 |
|---|---|---|
| ≤ 10~12 | O(n!) / O(2ⁿ·n) | 全排列、暴力回溯 |
| ≤ 20 | O(2ⁿ) | 子集枚举、状压 DP |
| ≤ 100~500 | O(n³) | 区间 DP、Floyd |
| ≤ 10³~10⁴ | O(n²) | 朴素 DP、双重循环 |
| ≤ 10⁵~10⁶ | O(n log n) / O(n) | 排序、二分、双指针、滑窗、哈希 |
| ≤ 10⁹ | O(log n) / O(√n) | 二分答案、数学 |

口诀：**10⁵ 往上基本告别 O(n²)**。

## 2. 哈希表套路

**识别信号**：要"查某元素是否出现过 / 出现几次 / 配对（a + b = target）/ 分组"。
**核心思想**：用 O(n) 空间换 O(1) 查询，把"两层循环找配对"降成一层。

```java
// 配对型：边遍历边查"我需要的另一半在不在"
Map<Integer, Integer> map = new HashMap<>(); // 值 -> 下标
for (int i = 0; i < nums.length; i++) {
    int need = target - nums[i];
    if (map.containsKey(need)) return new int[]{map.get(need), i};
    map.put(nums[i], i);
}

// 计数型
Map<Integer, Integer> cnt = new HashMap<>();
for (int x : nums) cnt.merge(x, 1, Integer::sum);

// 分组型：把"特征"作为 key
map.computeIfAbsent(key, k -> new ArrayList<>()).add(item);
```

## 3. 双指针套路

三种子型，看清楚是哪一种：

### (a) 对撞指针（左右往中间夹）
**信号**：数组**有序**，或求"两端配对"（盛水、两数之和 II、回文）。

```java
int l = 0, r = n - 1;
while (l < r) {
    if (条件) l++;
    else r--;
}
```

### (b) 快慢指针（同向不同速）
**信号**：原地移除/去重、链表找环、找中点。

```java
int slow = 0;                       // slow 指向"已处理好"区域的末尾
for (int fast = 0; fast < n; fast++) {
    if (需要保留 nums[fast]) {
        nums[slow++] = nums[fast];
    }
}
```

### (c) 固定一个 + 双指针（如三数之和）
**信号**：三数/四数之和。先排序，外层固定一个数，内层用对撞指针，并跳过重复。

```java
Arrays.sort(nums);
for (int i = 0; i < n; i++) {
    if (i > 0 && nums[i] == nums[i-1]) continue; // 去重
    int l = i + 1, r = n - 1;
    while (l < r) { ... }
}
```

## 4. 滑动窗口套路

**识别信号**：**连续**子数组/子串 + 求"最长/最短/是否存在满足某条件的窗口"。
**核心**：右指针扩张，破坏条件时左指针收缩，窗口内维护一个状态（计数/和）。

### 不定长窗口（最常见）通用模板

```java
int left = 0, ans = 0;
// 用某种结构维护窗口状态，如 Map / int[26] / sum
for (int right = 0; right < n; right++) {
    加入 nums[right] 到窗口;            // 扩张
    while (窗口不满足条件) {            // 收缩到重新满足
        移除 nums[left] 出窗口;
        left++;
    }
    ans = Math.max(ans, right - left + 1); // 在“满足”状态下更新答案
}
```

> 求**最长**：在 while 收缩到"满足"后更新答案。
> 求**最短**：在 while 内（满足时）更新答案，再继续收缩。

### 定长窗口

```java
for (int right = 0; right < n; right++) {
    加入 right;
    if (right - left + 1 > k) { 移除 left; left++; } // 维持长度 = k
    if (right - left + 1 == k) { 更新答案; }
}
```

## 5. 前缀和套路

**识别信号**：频繁求"区间 [i, j] 的和"，或"和为 K 的子数组个数"。
**核心**：`preSum[j] - preSum[i] = 区间 (i, j] 的和`。配合哈希可 O(n) 统计。

```java
// 一维前缀和：sum[i] = nums[0..i-1] 之和
int[] sum = new int[n + 1];
for (int i = 0; i < n; i++) sum[i + 1] = sum[i] + nums[i];
// 区间 [l, r] 之和 = sum[r + 1] - sum[l]

// "和为 K 的子数组个数" = 前缀和 + 哈希（把两数之和思想用在前缀和上）
Map<Integer, Integer> cnt = new HashMap<>();
cnt.put(0, 1);                 // 空前缀，和为 0 出现 1 次
int pre = 0, ans = 0;
for (int x : nums) {
    pre += x;
    ans += cnt.getOrDefault(pre - k, 0); // 找有多少个前缀和 = pre - k
    cnt.merge(pre, 1, Integer::sum);
}
```

## 6. 二分查找套路

**识别信号**：**有序**数组找数；或"求满足条件的最小/最大值"（二分答案）。
**关键**：想清楚区间是 `[l, r]` 闭还是 `[l, r)` 半开，循环条件和移动要配套。

### 模板一：找具体值

```java
int l = 0, r = n - 1;          // 闭区间 [l, r]
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (nums[mid] == target) return mid;
    else if (nums[mid] < target) l = mid + 1;
    else r = mid - 1;
}
return -1;
```

### 模板二：找左边界（第一个 ≥ target 的位置）

```java
int l = 0, r = n;              // 半开 [l, r)，r 可取 n
while (l < r) {
    int mid = l + (r - l) / 2;
    if (nums[mid] >= target) r = mid;   // 满足，往左继续找更小下标
    else l = mid + 1;
}
return l;                       // l == r，即第一个满足位置（可能 = n）
```

> 记住一句：**"找左边界 → 满足条件时收右边界 `r = mid`"**。找右边界对称处理。

## 7. 链表技巧

核心三件套：**虚拟头结点**、**快慢指针**、**反转**。

```java
// 1) 虚拟头：避免单独处理头结点（删除/合并必用）
ListNode dummy = new ListNode(0, head);
ListNode cur = dummy;
// ... 操作 ...
return dummy.next;

// 2) 反转链表（迭代）
ListNode prev = null, cur = head;
while (cur != null) {
    ListNode next = cur.next; // 暂存
    cur.next = prev;          // 反向
    prev = cur;               // 前移
    cur = next;
}
return prev;                  // 新头

// 3) 快慢指针找中点 / 找环
ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
}
// slow 即中点；找环时 slow==fast 则有环
```

## 8. 栈 / 单调栈

### 普通栈
**信号**：括号匹配、表达式求值、需要"最近一次"配对。

```java
Deque<Character> st = new ArrayDeque<>();
for (char c : s.toCharArray()) {
    if (是左括号) st.push(c);
    else { if (st.isEmpty() || !匹配) return false; st.pop(); }
}
return st.isEmpty();
```

### 单调栈
**信号**：求每个元素"下一个更大/更小元素"、柱状图最大矩形、接雨水。
**核心**：栈内保持单调（递增或递减），破坏单调性时弹出并结算。

```java
// 找“下一个更大元素”：维护单调递减栈，存下标
Deque<Integer> st = new ArrayDeque<>();
int[] res = new int[n];
for (int i = 0; i < n; i++) {
    while (!st.isEmpty() && nums[i] > nums[st.peek()]) {
        int idx = st.pop();
        res[idx] = nums[i];   // i 就是 idx 的下一个更大元素
    }
    st.push(i);
}
```

## 9. 单调队列

**信号**：**滑动窗口的最大值/最小值**。
**核心**：双端队列存下标，队头始终是窗口最大值的下标，保持队列单调递减。

```java
Deque<Integer> dq = new ArrayDeque<>(); // 存下标，对应值单调递减
for (int i = 0; i < n; i++) {
    while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) dq.pollLast(); // 维持递减
    dq.offerLast(i);
    if (dq.peekFirst() <= i - k) dq.pollFirst();   // 队头滑出窗口
    if (i >= k - 1) res[i - k + 1] = nums[dq.peekFirst()]; // 队头即窗口最大值
}
```

## 10. 堆 / Top K

**信号**：第 K 大/小、前 K 高频、合并 K 个有序、动态求中位数。

```java
// 第 K 大：维护大小为 k 的小根堆
PriorityQueue<Integer> pq = new PriorityQueue<>();
for (int x : nums) {
    pq.offer(x);
    if (pq.size() > k) pq.poll();   // 弹掉最小，留下最大的 k 个
}
return pq.peek();                   // 堆顶 = 第 K 大

// 前 K 高频：先用 map 计数，再用堆按频次取前 K
```

> 记忆：求**第 K 大用小根堆**（堆顶是这 k 个里最小的，即第 K 大）；求第 K 小用大根堆，反过来。

## 11. 树的遍历

### DFS 递归（前/中/后序）

```java
void dfs(TreeNode root) {
    if (root == null) return;
    // 前序：在这里处理 root
    dfs(root.left);
    // 中序：在这里处理 root（BST 中序 = 升序！）
    dfs(root.right);
    // 后序：在这里处理 root（需要先知道左右子树结果时用）
}
```

### BFS 层序（队列）

```java
Queue<TreeNode> q = new LinkedList<>();
q.offer(root);
while (!q.isEmpty()) {
    int size = q.size();            // 固定本层节点数
    for (int i = 0; i < size; i++) {
        TreeNode node = q.poll();
        // 处理 node
        if (node.left != null) q.offer(node.left);
        if (node.right != null) q.offer(node.right);
    }
    // 这里是“一层处理完”的时机（层序、右视图等）
}
```

**心法**：自顶向下传参数用前序；需要左右子树信息再合并用后序；按层处理用 BFS。

## 12. 回溯套路

**信号**：求**所有**方案（子集、排列、组合、切割、棋盘）。数据范围通常很小。
**核心**：决策树 + 做选择 → 递归 → 撤销选择。

```java
List<List<Integer>> res = new ArrayList<>();
void backtrack(int start, List<Integer> path) {
    // 1) 终止条件：收集结果
    res.add(new ArrayList<>(path));   // 注意存副本！

    // 2) 遍历选择
    for (int i = start; i < n; i++) {
        if (剪枝条件) continue;        // 去重 / 限制
        path.add(nums[i]);            // 做选择
        backtrack(i + 1, path);       // 递归（组合用 i+1，排列用 used[] 标记）
        path.remove(path.size() - 1); // 撤销选择（回溯）
    }
}
```

**三种典型**：
- **组合/子集**：用 `start` 控制不回头取。
- **排列**：用 `boolean[] used` 标记已用，每层从 0 开始。
- **去重**：先排序，`if (i > start && nums[i] == nums[i-1]) continue;`。

## 13. 图

### 网格 / 图的 DFS、BFS
和树类似，但**要标记 visited**（图有环），网格用方向数组。

```java
// 网格 DFS（岛屿问题）
void dfs(char[][] g, int i, int j) {
    if (i < 0 || i >= m || j < 0 || j >= n || g[i][j] != '1') return;
    g[i][j] = '0';                  // 标记已访问（或用单独 visited 数组）
    dfs(g, i+1, j); dfs(g, i-1, j); dfs(g, i, j+1); dfs(g, i, j-1);
}
```

### 拓扑排序（课程表）
**信号**：判断有向图能否完成（无环）、求依赖顺序。用 BFS（入度为 0 入队）。

### 并查集（连通性）
**信号**：动态合并、判两点是否连通、统计连通分量。

```java
int[] fa;
int find(int x) { return fa[x] == x ? x : (fa[x] = find(fa[x])); } // 路径压缩
void union(int a, int b) { fa[find(a)] = find(b); }
```

## 14. 动态规划套路

**识别信号**：求"方案数 / 最值 / 能否达成"，且**当前决策依赖之前的子问题结果**，有重叠子问题。

**五步法**：
1. **定义状态** `dp[i]` 或 `dp[i][j]` 表示什么（最难、最关键）。
2. **状态转移方程**：`dp[i]` 由哪些更小的状态推出。
3. **初始化** base case。
4. **遍历顺序**：保证算 `dp[i]` 时它依赖的状态已算好。
5. **返回值**：通常是 `dp[n]` 或整个表的最值。

**常见类型与转移**：

```java
// 线性 DP（爬楼梯/打家劫舍）
dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]);

// 最大子数组和（Kadane）
dp[i] = Math.max(nums[i], dp[i-1] + nums[i]);

// 最长递增子序列 LIS：O(n²)
for (int i = 0; i < n; i++)
  for (int j = 0; j < i; j++)
    if (nums[j] < nums[i]) dp[i] = Math.max(dp[i], dp[j] + 1);

// 0/1 背包（一维滚动，容量倒序）
for (int i = 0; i < items; i++)
  for (int c = W; c >= w[i]; c--)
    dp[c] = Math.max(dp[c], dp[c - w[i]] + v[i]);

// 二维 DP（编辑距离/路径）：dp[i][j] 依赖左、上、左上
```

**优化**：很多一维 DP 只依赖前一两个状态 → 用滚动变量把空间降到 O(1)。

## 15. 贪心

**信号**：每一步都做"局部最优"，且能证明局部最优导向全局最优（跳跃游戏、分发糖果、区间调度）。
**心法**：贪心难在**证明**，刷题时记住经典模型：
- **区间问题**：按右端点排序选不重叠区间；按左端点排序合并区间。
- **跳跃/覆盖**：维护"能到达的最远位置"。
- **两端分配**：从两个方向各扫一遍取 max（分发糖果）。
